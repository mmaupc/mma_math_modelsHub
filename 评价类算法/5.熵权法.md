# 熵权法

熵权法是多指标评价里非常经典的一种客观赋权方法。它的核心思想来自信息论：如果一个指标在不同方案之间差异很小，说明它提供的信息不多；如果一个指标差异很大，说明它更能区分方案，因此应该给更大的权重。

所以，熵权法本质上不是“谁更重要就给谁高权重”，而是“谁更有区分能力就给谁高权重”。

## 这个方法在解决什么问题

假设有 $m$ 个待评价对象、$n$ 个指标。经过正向化和标准化后，得到标准化矩阵

$$
Z = (z_{ij})_{m \times n}
$$

对第 $j$ 个指标，定义比例矩阵

$$
p_{ij} = \frac{z_{ij}}{\sum_{i=1}^m z_{ij}}
$$

再定义该指标的信息熵为

$$
e_j = -k \sum_{i=1}^m p_{ij} \ln p_{ij},
\quad
k = \frac{1}{\ln m}
$$

于是差异系数为

$$
d_j = 1 - e_j
$$

最后得到权重

$$
w_j = \frac{d_j}{\sum_{j=1}^n d_j}
$$

通常：

- $e_j$ 越大，说明该指标越“平均”、越“混乱”，区分能力越弱。
- $d_j$ 越大，说明该指标差异越明显，区分能力越强。
- $w_j$ 越大，说明该指标客观上更能拉开方案差距。

## 数学原理

### 1. 熵在这里表示“均匀程度”

在信息论里，熵通常用于刻画不确定性或混乱程度。

放到评价问题里，可以把它理解成：

- 如果一个指标上，各个方案数值都差不多，那么这个指标的信息分布很均匀，熵就高。
- 如果一个指标上，各个方案差异很明显，那么这个指标的信息更集中，熵就低。

这时我们会认为：差异更大的指标更有用，因为它更能把方案区分开。

### 2. 为什么要先标准化

熵权法的权重来源于“各方案在某指标上的相对比例”，所以如果原始数据量纲不同，就必须先做正向化和标准化，否则比例矩阵没有可比性。

常用的极差标准化为

$$
z_{ij} = \frac{x'_{ij} - \min_i x'_{ij}}{\max_i x'_{ij} - \min_i x'_{ij}}
$$

其中 $x'_{ij}$ 表示正向化后的数值。

### 3. 比例矩阵的意义

比例矩阵

$$
p_{ij} = \frac{z_{ij}}{\sum_i z_{ij}}
$$

不是在算“得分”，而是在看：第 $i$ 个方案在第 $j$ 个指标这一列里，占了多大比例。

如果一整列的比例都差不多，说明大家在这个指标上表现接近，这列对排序帮助有限。

### 4. 为什么权重和“差异”有关

熵权法不是在表达“政策上更重视什么”，而是在表达“数据本身更能区分什么”。

因此，它得到的是一种客观权重，而不是主观偏好权重。

这也是为什么熵权法经常和 AHP 结合使用：前者提供客观差异信息，后者提供决策偏好信息。

### 5. 零值问题怎么处理

因为公式里有 $\ln p_{ij}$，所以如果 $p_{ij} = 0$，就会遇到数值问题。

理论上，极限意义下有

$$
\lim_{p \to 0^+} p \ln p = 0
$$

实际编程中，通常有两种处理方式：

- 遇到 0 时直接按 0 处理。
- 在比例矩阵里加一个很小的正数，例如 $10^{-12}$。

## 一个直觉 example

假设你在比较 5 所学校，指标有两个：

- 指标 A：5 所学校的数据几乎一样。
- 指标 B：5 所学校差异很大。

直觉上，指标 A 对排名帮助不大，因为它很难把学校拉开；指标 B 更能区分学校，所以更“有信息量”。

熵权法正是把这种直觉量化了：

- 差异小，熵高，权重低。
- 差异大，熵低，权重高。

## 标准流程

### 第一步：做正向化

把所有指标都处理成“越大越好”。

### 第二步：做标准化

将不同量纲映射到同一尺度。

### 第三步：计算比例矩阵

按列归一化，得到 $p_{ij}$。

### 第四步：计算熵值和差异系数

分别求出 $e_j$ 和 $d_j = 1 - e_j$。

### 第五步：得到权重

将差异系数归一化得到 $w_j$。

### 第六步：做后续综合评价

熵权法本身主要解决“权重怎么来”，得到权重后，通常还要再接加权求和法、TOPSIS 等模型完成最终排序。

## 建模案例：城市公共服务绩效评价

### 问题背景

现有 5 个城市，需要从公共服务绩效角度进行综合评价。给出的指标如下：

- 每千人执业医生数：越大越好。
- 公交站点覆盖率：越大越好。
- 污水处理率：越大越好。
- 人均图书馆藏书量：越大越好。
- 每万人投诉量：越小越好。

由于题目没有给出权重，希望使用熵权法从数据中客观确定权重，再利用加权求和法做排序。

### 建模思路

这里要分清楚两步：

1. 用熵权法算权重。
2. 用加权求和法做综合得分。

很多初学者会把这两件事混成一步，这是一个典型误区。

### Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "城市": ["A市", "B市", "C市", "D市", "E市"],
        "每千人执业医生数": [3.4, 2.8, 3.7, 3.1, 3.5],
        "公交站点覆盖率": [82, 76, 88, 80, 85],
        "污水处理率": [94, 91, 96, 90, 95],
        "人均图书馆藏书量": [1.5, 1.2, 1.8, 1.1, 1.6],
        "每万人投诉量": [18, 22, 15, 24, 17],
    }
)

indicator_types = {
    "每千人执业医生数": "benefit",
    "公交站点覆盖率": "benefit",
    "污水处理率": "benefit",
    "人均图书馆藏书量": "benefit",
    "每万人投诉量": "cost",
}


def minmax_to_01(series, indicator_type):
    series = series.astype(float)
    max_val = series.max()
    min_val = series.min()

    if np.isclose(max_val, min_val):
        return pd.Series(np.ones(len(series)), index=series.index)

    if indicator_type == "benefit":
        return (series - min_val) / (max_val - min_val)
    if indicator_type == "cost":
        return (max_val - series) / (max_val - min_val)

    raise ValueError("未知指标类型")


normalized = pd.DataFrame(index=data.index)
for col, indicator_type in indicator_types.items():
    normalized[col] = minmax_to_01(data[col], indicator_type)

# 避免出现 log(0)
safe_normalized = normalized + 1e-12
p = safe_normalized / safe_normalized.sum(axis=0)

m = len(data)
k = 1 / np.log(m)
entropy = -k * (p * np.log(p)).sum(axis=0)
divergence = 1 - entropy
weights = divergence / divergence.sum()

score = normalized.mul(weights, axis=1).sum(axis=1)

result = data[["城市"]].copy()
result["综合得分"] = score
result = result.sort_values("综合得分", ascending=False).reset_index(drop=True)

print("标准化矩阵：")
print(normalized.round(4))

print("\n熵值：")
print(entropy.round(4))

print("\n熵权：")
print(weights.round(4))

print("\n综合排序结果：")
print(result.round(4))
```

### 结果解读

运行后你会得到：

- 每个指标的熵值 $e_j$。
- 每个指标的差异系数 $d_j$。
- 最终客观权重 $w_j$。
- 基于熵权的综合排序结果。

如果某个指标在 5 个城市之间差异特别小，那么它的熵值往往会更接近 1，最终权重也会更低。反过来，如果某个指标拉开了明显差距，它的权重通常会更高。

这说明熵权法更重视“区分能力强的指标”。

## 熵权法的优缺点与适用场景

### 优点

- 完全由数据决定，客观性较强。
- 计算逻辑清晰，容易实现。
- 很适合和 TOPSIS、加权求和法结合。
- 当题目没有给权重时，常作为默认客观赋权工具。

### 局限

- 它反映的是“差异大小”，不是“决策偏好”。
- 如果某个指标差异大只是因为异常值，也可能被赋予较高权重。
- 若数据质量差、噪声大，熵权结果会受影响。
- 指标差异小不代表它不重要，只是代表它在当前样本里不容易区分方案。

### 适用场景

- 权重未知，希望由数据客观生成。
- 样本之间存在一定差异。
- 希望减少主观赋权带来的争议。

## 常见误区

### 1. 以为熵权法算出来的就是“绝对重要性”

不是。它衡量的是样本差异带来的信息量，不是政策或专家意义上的重要性。

### 2. 只算权重，不做后续排序

熵权法通常只是前半段，后面还要配合综合评价方法。

### 3. 忽略异常值

如果某列因为极端异常值被拉得很散，熵权可能被扭曲，所以建模前最好先做异常值检查。

## 一句话总结

熵权法是在回答“哪些指标更能把方案区分开”。它适合解决权重未知的问题，但它给出的是客观差异权重，而不是主观偏好权重。
