# PROMETHEE法

## 简介


PROMETHEE 是一类基于两两比较的多指标决策方法，完整名称为：

$$
\text{Preference Ranking Organization Method for Enrichment Evaluation}
$$

和“先把所有指标压成一个总分”不同，PROMETHEE 的思路是：

1. 先比较方案 $A$ 相对方案 $B$，到底优了多少；
2. 再比较方案 $A$ 相对方案 $C$，又优了多少；
3. 最后把这种“成对对决”的结果汇总起来。

所以，它特别像一个循环赛积分榜：

- 每个方案都和其他方案打一轮；
- 不仅看输赢，还看赢得是否明显；
- 最后根据净优势决定排名。

---

## 它在解决什么问题

假设有 $m$ 个方案、$n$ 个指标，评价矩阵记为

$$
G = (g_{ij})_{m \times n}.
$$

PROMETHEE 想解决的问题是：

> 当“优于”不是非黑即白，而是需要考虑差距大小时，怎样把多指标决策转化成一套有解释力的排序结果？

这类问题很常见。比如：

- 方案 A 比方案 B 高 1 分，到底算不算“明显更优”？
- 如果只高一点点，我们是否应该当作两者差不多？
- 如果高很多，是否应该认定 A 对 B 有强偏好？

PROMETHEE 的价值就在于：它允许我们把这种“差一点不算、差很多才算明显领先”的判断，显式写进模型。

---

## 数学原理

### 1. 差值而不是总分

设已经把所有指标统一处理成“越大越好”。对任意两个方案 $a$ 和 $b$，在第 $j$ 个指标上的差值定义为

$$
d_j(a,b) = g_j(a) - g_j(b).
$$

如果 $d_j(a,b)$ 很大，说明在指标 $j$ 上，方案 $a$ 明显优于方案 $b$。

但 PROMETHEE 不会直接把这个差值拿来用，而是进一步通过**偏好函数**把它映射到 $[0,1]$ 区间。

### 2. 偏好函数

记偏好函数为 $P_j(\cdot)$，则

$$
P_j(a,b) = P_j\bigl(d_j(a,b)\bigr).
$$

最简单的理解是：

- $P_j(a,b)=0$，表示在指标 $j$ 上，$a$ 并没有明显优于 $b$；
- $P_j(a,b)=1$，表示在指标 $j$ 上，$a$ 明显优于 $b$；
- 中间值表示“有一定偏好，但还没有强到完全压制”。

### 3. 一个常用的线性偏好函数

初学时最常见的是带阈值的线性偏好函数。设：

- $q_j$ 为无差异阈值；
- $p_j$ 为严格偏好阈值，且 $p_j > q_j$。

则可定义

$$
P_j(a,b)=
\begin{cases}
0, & d_j(a,b)\le q_j,\\
\dfrac{d_j(a,b)-q_j}{p_j-q_j}, & q_j<d_j(a,b)<p_j,\\
1, & d_j(a,b)\ge p_j.
\end{cases}
$$

这个式子的含义很自然：

- 如果只领先一点点，不足以说明明显更优，就记为 $0$；
- 如果领先非常多，就记为 $1$；
- 如果介于两者之间，就按比例给一个偏好程度。

### 4. 综合偏好指数

有了各指标上的偏好值之后，就可以加权得到方案 $a$ 相对方案 $b$ 的综合偏好指数：

$$
\pi(a,b) = \sum_{j=1}^n w_j P_j(a,b),
$$

其中

$$
w_j \ge 0, \qquad \sum_{j=1}^n w_j = 1.
$$

$\pi(a,b)$ 越大，说明从整体看，$a$ 越明显优于 $b$。

### 5. 正偏好流、负偏好流与净偏好流

对每个方案 $a$，定义：

正偏好流

$$
\phi^+(a) = \frac{1}{m-1}\sum_{b\ne a}\pi(a,b),
$$

负偏好流

$$
\phi^-(a) = \frac{1}{m-1}\sum_{b\ne a}\pi(b,a),
$$

净偏好流

$$
\phi(a) = \phi^+(a) - \phi^-(a).
$$

它们的意义分别是：

- $\phi^+(a)$ 大：说明方案 $a$ 经常能压过别人；
- $\phi^-(a)$ 小：说明方案 $a$ 不太容易被别人压过；
- $\phi(a)$ 大：说明方案 $a$ 的净优势更强。

---

## PROMETHEE I 和 PROMETHEE II

这两个名字很容易混。

### PROMETHEE I

同时看 $\phi^+$ 和 $\phi^-$，得到的是**部分排序**。有时两个方案可能互有优劣，不一定能直接比较出绝对先后。

### PROMETHEE II

用净偏好流

$$
\phi(a) = \phi^+(a) - \phi^-(a)
$$

做完整排序，得到的是**全排序**。

教学和比赛中最常用的就是 PROMETHEE II，因为它更容易直接给出排名。下面也主要按这个版本讲。

---

## 一个能手算的直觉例子

假设我们比较两个课题 A 和 B，在“预期影响”这个指标上：

- A 的得分是 86；
- B 的得分是 84。

如果我们设

$$
q = 3, \qquad p = 10,
$$

那么两者差值为

$$
d(A,B)=86-84=2.
$$

因为

$$
2 \le q,
$$

所以这一项的偏好值为

$$
P(A,B)=0.
$$

也就是说，虽然 A 的分数略高于 B，但差距太小，不足以被认定为“明显更优”。

如果换成：

- A 的得分是 95；
- B 的得分是 84。

那么

$$
d(A,B)=11 \ge p,
$$

于是

$$
P(A,B)=1.
$$

这表示 A 在这个指标上对 B 形成了明确优势。

这个例子说明了 PROMETHEE 的精髓：

**不是所有领先都算领先，要看领先得够不够明显。**

## 标准流程

PROMETHEE II 一般按下面 6 步进行：

### 第一步：统一指标方向

把所有指标处理成“越大越好”。

### 第二步：选偏好函数与阈值

常见偏好函数包括：

- 普通型；
- U 型；
- V 型；
- 线性型；
- 高斯型等。

初学时最常见的是带阈值的线性型。

### 第三步：计算两两差值与单指标偏好值

$$
d_j(a,b)=g_j(a)-g_j(b),
$$

$$
P_j(a,b)=P_j\bigl(d_j(a,b)\bigr).
$$

### 第四步：计算综合偏好指数

$$
\pi(a,b)=\sum_{j=1}^n w_j P_j(a,b).
$$

### 第五步：计算偏好流

$$
\phi^+(a),\quad \phi^-(a),\quad \phi(a).
$$

### 第六步：排序并解释结果

对 PROMETHEE II，通常按 $\phi(a)$ 从大到小排序。

## 案例讲解：科研课题立项优先级排序

### 问题背景

某实验室收到 4 个备选科研课题，希望确定优先立项顺序。评价指标包括：

- 预期影响：越大越好；
- 实施成本：越小越好；
- 技术风险：越小越好；
- 创新性：越大越好；
- 团队匹配度：越大越好。

实验室不想直接做“加权总分”，因为他们更关心：

> 某个课题相对于另一个课题，到底强多少？这种优势是否足够明显？

这正是 PROMETHEE 的强项。

---

## 可运行 Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "课题": ["T1", "T2", "T3", "T4"],
        "预期影响": [88, 82, 91, 85],
        "实施成本": [45, 38, 52, 41],
        "技术风险": [0.32, 0.24, 0.40, 0.28],
        "创新性": [84, 79, 92, 86],
        "团队匹配度": [87, 90, 80, 88],
    }
)

weights = pd.Series(
    {
        "预期影响": 0.28,
        "实施成本": 0.18,
        "技术风险": 0.18,
        "创新性": 0.22,
        "团队匹配度": 0.14,
    }
)
weights = weights / weights.sum()

indicator_types = {
    "预期影响": "benefit",
    "实施成本": "cost",
    "技术风险": "cost",
    "创新性": "benefit",
    "团队匹配度": "benefit",
}


def positive_transform(df, indicator_types):
    transformed = df.copy()
    for col, col_type in indicator_types.items():
        if col_type == "cost":
            transformed[col] = transformed[col].max() - transformed[col] + transformed[col].min()
    return transformed


def linear_preference(diff, q, p):
    if diff <= q:
        return 0.0
    if diff >= p:
        return 1.0
    return (diff - q) / (p - q)


X = positive_transform(data[list(indicator_types.keys())], indicator_types)
ranges = X.max(axis=0) - X.min(axis=0)

# 教学示例里，用极差的一部分构造阈值；真实项目里更建议由专家给出
q = 0.10 * ranges
p = 0.40 * ranges
p = p.where(p > q, q + 1e-12)

names = data["课题"].tolist()
m = len(names)
pi = pd.DataFrame(np.zeros((m, m)), index=names, columns=names)

for i in range(m):
    for j in range(m):
        if i == j:
            continue
        pref = 0.0
        for col in weights.index:
            diff = X.iloc[i][col] - X.iloc[j][col]
            pref += weights[col] * linear_preference(diff, q[col], p[col])
        pi.iloc[i, j] = pref

phi_plus = pi.sum(axis=1) / (m - 1)
phi_minus = pi.sum(axis=0) / (m - 1)
phi = phi_plus - phi_minus

result = pd.DataFrame(
    {
        "课题": names,
        "phi_plus": phi_plus,
        "phi_minus": phi_minus,
        "phi": phi,
    }
).sort_values("phi", ascending=False).reset_index(drop=True)

print("正向化后的数据：")
print(X.round(4))
print("\n无差异阈值 q：")
print(q.round(4))
print("\n严格偏好阈值 p：")
print(p.round(4))
print("\n综合偏好矩阵：")
print(pi.round(4))
print("\nPROMETHEE II 排序结果：")
print(result.round(4))
```

## 结果应该怎么解释

### 1. 综合偏好矩阵 $\pi(a,b)$

它表示“方案 $a$ 相对方案 $b$ 的整体偏好强度”。

- 越接近 1，说明 $a$ 越明显优于 $b$；
- 越接近 0，说明 $a$ 对 $b$ 没什么明显优势。

### 2. 正偏好流 $\phi^+$

它表示这个方案平均能压过别人多少。

### 3. 负偏好流 $\phi^-$

它表示这个方案平均会被别人压过多少。

### 4. 净偏好流 $\phi$

它是最终排序最常用的指标：

- $\phi$ 越大，说明方案总体净优势越强；
- 排名越靠前，说明该方案在成对比较中更占优。

---

## PROMETHEE 和 ELECTRE 的区别

这两个方法都属于 outranking 思路，也就是“超越关系”方法，但风格不一样。

### PROMETHEE 更平滑

它通过偏好函数把差值映射成连续的偏好程度，因此更适合得到一个平滑、可解释的排序结果。

### ELECTRE 更强调否决与筛选

ELECTRE 会更强调“一致性”和“否决性”，如果某个方案虽然多数指标占优，但在某个关键指标上差得太多，可能会直接被挡掉。

因此：

- 想做全排序，PROMETHEE 往往更顺手；
- 想做带底线约束的筛选，ELECTRE 往往更合适。

---

## 优缺点与适用场景

### 优点

- 很适合处理两两偏好比较；
- 可以显式表达“差多少才算明显更优”；
- 结果解释性好；
- 比单纯总分法更贴近真实决策思路。

### 局限

- 需要设定偏好函数和阈值；
- 样本很多时，两两比较会增加计算量；
- 初学者容易在阈值选择上没有感觉；
- 阈值设置不合理会明显影响结论。

### 适用场景

- 项目优先级排序；
- 供应商评价；
- 科研立项；
- 更强调“成对优势”而不是机械总分的决策问题。

---

## 一句话总结

PROMETHEE 的本质，是让每个方案都去和别人打一轮，再根据“净胜优势”完成排序。
