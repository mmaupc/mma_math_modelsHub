# DEMATEL法

## 简介

DEMATEL（Decision Making Trial and Evaluation Laboratory）是一种分析**因素之间因果关系**的方法。

它和一般综合评价方法最不一样的地方在于：

- 普通权重法更关心“谁更重要”；
- DEMATEL 更进一步关心“谁在影响谁”。

如果你的问题里，指标之间明显相互作用，例如：

- 政策支持影响研发投入；
- 研发投入影响成果转化；
- 企业活力又反过来影响研发投入；

那 DEMATEL 往往就非常有用。

---

## 它在解决什么问题

很多复杂系统中的因素都不是孤立的。

例如在区域创新系统里：

- 人才储备会影响研发投入；
- 政策支持会影响企业活力；
- 企业活力又会影响成果转化；
- 成果转化反过来还可能吸引更多人才。

这类问题如果只用普通评分法，最多告诉你“哪些因素重要”，但回答不了下面这些更关键的问题：

- 哪些因素是根源；
- 哪些因素是结果；
- 哪些因素是系统中的“发动机”；
- 哪些因素只是被动承接影响。

DEMATEL 正是为了解决这种因果结构分析问题。

---

## 数学原理

### 1. 直接影响矩阵

设有 $n$ 个因素，专家给出直接影响矩阵

$$
A = (a_{ij})_{n\times n},
$$

其中：

- $a_{ij}$ 表示因素 $i$ 对因素 $j$ 的直接影响强度；
- 通常有

$$
a_{ii}=0.
$$

例如：

- $a_{ij}=0$ 表示没有直接影响；
- 值越大表示影响越强。

### 2. 为什么要标准化

为了保证后续矩阵级数收敛，需要先把 $A$ 标准化。

一种常见做法是：

$$
s = \max\left(\max_i \sum_{j=1}^n a_{ij},\ \max_j \sum_{i=1}^n a_{ij}\right),
$$

$$
N = \frac{A}{s}.
$$

标准化后，$N$ 的数值会缩小，从而保证后面的

$$
(I-N)^{-1}
$$

可以稳定存在。

### 3. 总影响矩阵

DEMATEL 最核心的矩阵是总影响矩阵：

$$
T = N(I-N)^{-1}.
$$

这个式子可以展开为

$$
T = N + N^2 + N^3 + \cdots
$$

这一步很关键，因为它说明：

- $N$ 表示直接影响；
- $N^2$ 表示通过一个中间因素产生的间接影响；
- $N^3$ 表示更长链条上的间接影响；
- $T$ 则把直接影响和所有间接影响都加总起来了。

这就是 DEMATEL 的灵魂。

### 4. 行和与列和

对总影响矩阵 $T=(t_{ij})$，定义：

$$
D_i = \sum_{j=1}^n t_{ij},
$$

$$
C_i = \sum_{j=1}^n t_{ji}.
$$

其中：

- $D_i$ 表示因素 $i$ 对其他因素施加的总影响；
- $C_i$ 表示因素 $i$ 从其他因素受到的总影响。

### 5. 中心度与原因度

进一步定义：

中心度

$$
D_i + C_i,
$$

原因度

$$
D_i - C_i.
$$

它们分别表示：

- **中心度**：因素在整个系统中有多重要；
- **原因度**：因素更像“原因”还是“结果”。

解释上通常是：

- 若

$$
D_i - C_i > 0,
$$

则该因素偏原因型；

- 若

$$
D_i - C_i < 0,
$$

则该因素偏结果型。

---

## 为什么 $T=N+N^2+N^3+\cdots$ 很重要

这一步是 DEMATEL 和很多普通方法拉开差距的地方。

举个简单例子：

- 政策支持会影响研发投入；
- 研发投入会影响成果转化。

那么即使“政策支持”没有特别强的直接路径指向“成果转化”，它仍然会通过

$$
\text{政策支持} \to \text{研发投入} \to \text{成果转化}
$$

这条链条产生间接影响。

也就是说：

**DEMATEL 不是只看眼前的直接作用，而是把整个系统里的传导效应也考虑进来了。**

---

## 一个能手算的直觉例子

假设有 3 个因素：

- $F_1$：政策支持；
- $F_2$：研发投入；
- $F_3$：成果转化。

直接影响关系可简化成：

- 政策支持强烈影响研发投入；
- 研发投入强烈影响成果转化；
- 政策支持直接影响成果转化较弱。

那么在直接影响矩阵里，你看到的可能只是：

$$
F_1 \to F_2 \text{ 强},\qquad F_2 \to F_3 \text{ 强},\qquad F_1 \to F_3 \text{ 弱}.
$$

但 DEMATEL 在计算总影响矩阵后，会把

$$
F_1 \to F_2 \to F_3
$$

这条间接路径也算进去，所以最终你会发现：

- 政策支持对成果转化的总影响并不小；
- 这就是系统传导效应。

---

## 标准流程

DEMATEL 通常按下面 6 步进行：

### 第一步：确定因素集合

明确系统中要分析的核心因素。

### 第二步：构造直接影响矩阵

由专家评分给出因素之间的直接影响程度。

### 第三步：标准化

把直接影响矩阵转化为标准化矩阵 $N$。

### 第四步：求总影响矩阵

$$
T = N(I-N)^{-1}.
$$

### 第五步：计算 $D$、$C$、中心度、原因度

用于判断每个因素在系统中的位置。

### 第六步：做因果解释或绘制关系图

在论文和报告中，常根据阈值进一步画出因果关系图。

---

## 案例讲解：区域创新生态的因果诊断

### 问题背景

某研究团队想分析区域创新生态系统中 5 个核心因素之间的关系：

- 人才储备；
- 研发投入；
- 政策支持；
- 企业活力；
- 成果转化。

他们不只是想知道“谁重要”，更想知道：

> 谁在驱动系统变化，谁更多是结果承接者？

因此采用 DEMATEL。

---

## 可运行 Python 代码

下面示例除了计算总影响矩阵、中心度、原因度，还会额外给出一个简化的“强关系边”列表，方便理解因果网络。

```python
import numpy as np
import pandas as pd


factors = ["人才储备", "研发投入", "政策支持", "企业活力", "成果转化"]

A = np.array(
    [
        [0, 3, 2, 2, 3],
        [2, 0, 2, 3, 4],
        [3, 3, 0, 2, 3],
        [2, 3, 1, 0, 4],
        [1, 2, 1, 2, 0],
    ],
    dtype=float,
)

# 标准化
s = max(A.sum(axis=1).max(), A.sum(axis=0).max())
N = A / s

# 总影响矩阵
I = np.eye(len(factors))
T = N @ np.linalg.inv(I - N)

# 行和 D、列和 C
D = T.sum(axis=1)
C = T.sum(axis=0)
centrality = D + C
causality = D - C

result = pd.DataFrame(
    {
        "因素": factors,
        "D": D,
        "C": C,
        "中心度(D+C)": centrality,
        "原因度(D-C)": causality,
    }
).sort_values("中心度(D+C)", ascending=False)

print("总影响矩阵 T：")
print(pd.DataFrame(T, index=factors, columns=factors).round(4))

print("\n中心度与原因度：")
print(result.reset_index(drop=True).round(4))

cause_group = result.loc[result["原因度(D-C)"] > 0, "因素"].tolist()
effect_group = result.loc[result["原因度(D-C)"] <= 0, "因素"].tolist()

print("\n原因型因素：", cause_group)
print("结果型因素：", effect_group)

# 取非对角元素均值作为简化阈值，提取“较强影响关系”
T_df = pd.DataFrame(T, index=factors, columns=factors)
mask = ~np.eye(len(factors), dtype=bool)
threshold = T_df.to_numpy()[mask].mean()

edges = []
for i, src in enumerate(factors):
    for j, dst in enumerate(factors):
        if i != j and T[i, j] >= threshold:
            edges.append((src, dst, T[i, j]))

edge_df = pd.DataFrame(edges, columns=["起点因素", "终点因素", "影响强度"]).sort_values(
    "影响强度", ascending=False
)

print(f"\n关系阈值: {threshold:.4f}")
print("\n较强影响关系：")
print(edge_df.round(4))
```

## 结果应该怎么解释

### 1. 总影响矩阵 $T$

它告诉你每个因素对其他因素的**总影响**，其中已经包含：

- 直接影响；
- 间接影响。

### 2. 中心度 $D+C$

中心度越大，说明该因素在整个系统里越关键。它可能既强烈影响别人，也深受别人影响。

### 3. 原因度 $D-C$

- 原因度大于 0：更偏“原因型”因素；
- 原因度小于 0：更偏“结果型”因素。

所以原因度其实在回答：

> 这个因素更像驱动器，还是更像结果端？

### 4. 强关系边

通过设定阈值，我们还可以把影响特别强的路径提取出来，形成一个简化版因果网络。

这对于写论文里的机制分析部分非常有帮助。

---

## DEMATEL 和 ANP / ISM 的关系

DEMATEL 经常和其他结构方法一起使用。

### 和 ANP 的关系

DEMATEL 可以先识别因素之间的影响关系，再为 ANP 提供网络结构基础。

### 和 ISM 的关系

DEMATEL 擅长“影响强度和原因结果”的分析，ISM 更擅长“层级结构”的表达。

因此在很多研究里，会看到：

- DEMATEL + ANP；
- DEMATEL + ISM；
- DEMATEL + MICMAC 等组合。

---

## 优缺点与适用场景

### 优点

- 能识别因素之间的因果结构；
- 不只是给出重要性，还能区分原因型与结果型因素；
- 很适合做机制分析和政策解释；
- 可与 ANP、ISM 等方法联用。

### 局限

- 依赖专家对影响强度的判断；
- 直接影响矩阵有一定主观性；
- 更偏结构诊断，不直接给方案排序；
- 因素太多时，矩阵和关系图会变复杂。

### 适用场景

- 因素之间存在明显相互作用；
- 需要找系统中的关键驱动因子；
- 想从“评价问题”进一步走向“机制解释”；
- 政策分析、创新系统、风险传导、管理机制研究等问题。

---

## 一句话总结

DEMATEL 的核心价值，不只是告诉你“谁重要”，更是告诉你“谁在驱动系统变化，谁主要在承接结果”。
