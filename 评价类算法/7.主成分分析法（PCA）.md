# 主成分分析法（PCA）

主成分分析法（Principal Component Analysis, PCA）是降维方法里最常见的一种。它的核心想法是：如果很多指标彼此相关，那么就没必要把它们原封不动全部保留，而是可以重新组合出少数几个“主成分”，用更少的变量概括大部分信息。

在评价问题中，PCA 常被用来做两件事：

- 压缩指标，避免信息重复。
- 构造综合得分，对对象进行排序。

## 这个方法在解决什么问题

假设标准化后的数据矩阵为

$$
Z = (z_{ij})_{m \times n}
$$

PCA 希望构造新的变量

$$
F_1, F_2, \dots, F_n
$$

其中第 $k$ 个主成分可以写成原始变量的线性组合

$$
F_k = a_{1k} z_1 + a_{2k} z_2 + \cdots + a_{nk} z_n
$$

并满足：

- $F_1$ 的方差最大。
- 在与 $F_1$ 不相关的条件下，$F_2$ 的方差尽可能大。
- 依次类推。

换句话说，PCA 想找的是几条最能保留信息的“新坐标轴”。

## 数学原理

### 1. 为什么要先标准化

PCA 本质上在比较方差大小。若不同指标量纲差异太大，数值尺度大的变量会主导结果，所以通常先做标准化：

$$
z_{ij} = \frac{x_{ij} - \bar x_j}{s_j}
$$

如果原始指标里同时有收益型和成本型指标，也应先做正向化。

### 2. 相关矩阵或协方差矩阵

标准化后，通常计算相关系数矩阵

$$
R = \frac{1}{m-1} Z^\top Z
$$

然后对 $R$ 做特征值分解：

$$
R a_k = \lambda_k a_k
$$

其中：

- $\lambda_k$ 是第 $k$ 个特征值。
- $a_k$ 是对应特征向量。

### 3. 特征值为什么重要

第 $k$ 个主成分的方差等于对应的特征值 $\lambda_k$。因此：

- $\lambda_k$ 越大，说明这个主成分承载的信息越多。
- 主成分贡献率可写为

$$
\alpha_k = \frac{\lambda_k}{\sum_{j=1}^n \lambda_j}
$$

- 累积贡献率为

$$
\sum_{k=1}^q \alpha_k
$$

通常会选择前 $q$ 个主成分，使累积贡献率达到 80% 或 85% 以上。

### 4. 主成分得分

如果取前 $q$ 个主成分，则第 $i$ 个对象在第 $k$ 个主成分上的得分为

$$
F_{ik} = z_i^\top a_k
$$

进一步地，可以按贡献率加权构造综合得分：

$$
F_i = \sum_{k=1}^q \tilde \alpha_k F_{ik}
$$

其中

$$
\tilde \alpha_k = \frac{\lambda_k}{\sum_{r=1}^q \lambda_r}
$$

### 5. PCA 的本质不是“直接赋权”

这点非常重要。PCA 更像是在做“信息压缩”和“结构提取”，不是像熵权法那样直接对原指标给权重。

虽然最后常常会写出一个综合得分，但这个得分其实是先做了变量变换，再对主成分加权，而不是直接对原始指标线性打分。

## 一个直觉 example

假设你想评价学生体测表现，指标有：

- 50 米成绩。
- 1000 米成绩。
- 肺活量。
- 爆发力。
- 心肺耐力。

这些指标彼此并不是完全独立的。比如爆发力强的人，往往短跑也更强；心肺能力强的人，中长跑和肺活量往往也更好。

PCA 的思路就是：

- 不必机械保留所有原指标。
- 可以把它们重新组合成少数几个“身体素质综合因子”。

这就是“降维”的直觉。

## 标准流程

### 第一步：指标预处理

对成本型指标先正向化，再做标准化。

### 第二步：计算相关矩阵

检查变量之间是否存在较强相关性。

### 第三步：做特征值分解

得到特征值、特征向量、贡献率。

### 第四步：选择主成分个数

可参考累计贡献率、碎石图或特征值大于 1 的准则。

### 第五步：计算主成分得分和综合得分

根据主成分得分对样本做排序和解释。

## 建模案例：城市高质量发展综合评价

### 问题背景

现有 8 个城市，希望从高质量发展角度做综合评价，指标包括：

- 人均 GDP：越大越好。
- 研发投入强度：越大越好。
- 高技术就业占比：越大越好。
- 每万人专利授权量：越大越好。
- 单位 GDP 能耗：越小越好。
- 绿化覆盖率：越大越好。

由于这些指标之间可能存在较强相关性，因此使用 PCA 进行降维和综合评价。

### 建模思路

这个例子很适合 PCA：

- `人均 GDP`、`研发投入强度`、`专利授权量` 很可能同向变化。
- `单位 GDP 能耗` 与发展质量可能呈反向关系。
- 我们希望用少数几个综合因子来概括城市发展水平。

### Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "城市": ["A市", "B市", "C市", "D市", "E市", "F市", "G市", "H市"],
        "人均GDP": [16.8, 14.5, 18.2, 12.9, 17.5, 13.8, 19.0, 15.2],
        "研发投入强度": [3.2, 2.8, 3.8, 2.3, 3.5, 2.6, 4.1, 3.0],
        "高技术就业占比": [18.5, 16.2, 20.1, 14.8, 19.4, 15.5, 21.3, 17.0],
        "每万人专利授权量": [42, 35, 48, 30, 45, 33, 52, 38],
        "单位GDP能耗": [0.52, 0.58, 0.49, 0.63, 0.51, 0.60, 0.47, 0.56],
        "绿化覆盖率": [44, 41, 46, 39, 45, 40, 47, 42],
    }
)

indicator_types = {
    "人均GDP": "benefit",
    "研发投入强度": "benefit",
    "高技术就业占比": "benefit",
    "每万人专利授权量": "benefit",
    "单位GDP能耗": "cost",
    "绿化覆盖率": "benefit",
}


def positive_transform(series, indicator_type):
    series = series.astype(float)
    max_val = series.max()
    min_val = series.min()

    if np.isclose(max_val, min_val):
        return pd.Series(np.ones(len(series)), index=series.index)

    if indicator_type == "benefit":
        return series
    if indicator_type == "cost":
        return max_val - series

    raise ValueError("未知指标类型")


processed = pd.DataFrame(index=data.index)
for col, indicator_type in indicator_types.items():
    processed[col] = positive_transform(data[col], indicator_type)

Z = (processed - processed.mean()) / processed.std(ddof=1)
R = Z.corr()

eigvals, eigvecs = np.linalg.eigh(R.to_numpy())
idx = np.argsort(eigvals)[::-1]
eigvals = eigvals[idx]
eigvecs = eigvecs[:, idx]

explained_ratio = eigvals / eigvals.sum()
cumulative_ratio = explained_ratio.cumsum()
k = np.argmax(cumulative_ratio >= 0.85) + 1

scores = Z.to_numpy() @ eigvecs[:, :k]
loadings = eigvecs[:, :k] * np.sqrt(eigvals[:k])

# 统一主成分方向，便于解释
for j in range(k):
    if loadings[:, j].sum() < 0:
        loadings[:, j] *= -1
        scores[:, j] *= -1

component_weights = explained_ratio[:k] / explained_ratio[:k].sum()
composite_score = scores @ component_weights

eigen_table = pd.DataFrame(
    {
        "特征值": eigvals,
        "贡献率": explained_ratio,
        "累计贡献率": cumulative_ratio,
    },
    index=[f"PC{i+1}" for i in range(len(eigvals))],
)

loading_df = pd.DataFrame(
    loadings,
    index=processed.columns,
    columns=[f"PC{i+1}" for i in range(k)],
)

score_df = pd.DataFrame(
    scores,
    index=data["城市"],
    columns=[f"PC{i+1}得分" for i in range(k)],
)

result = pd.DataFrame(
    {
        "城市": data["城市"],
        "综合得分": composite_score,
    }
).sort_values("综合得分", ascending=False)

print("特征值与贡献率：")
print(eigen_table.round(4))

print("\n主成分载荷矩阵：")
print(loading_df.round(4))

print("\n各城市主成分得分：")
print(score_df.round(4))

print("\n综合排序结果：")
print(result.reset_index(drop=True).round(4))
```

### 结果解读

运行后要重点看三块内容：

- 特征值和累计贡献率，用来决定保留几个主成分。
- 载荷矩阵，用来解释每个主成分的含义。
- 主成分得分和综合得分，用来排序城市。

通常会出现这样的解释方式：

- 第一主成分在 `人均 GDP`、`研发投入强度`、`专利授权量` 上载荷都较高，可以理解为“创新与经济综合发展因子”。
- 第二主成分可能更偏向“绿色与环境质量因子”。

这说明 PCA 的价值不仅在于排序，还在于帮你看清“这些指标背后其实在讲哪几件大事”。

## PCA 的优缺点与适用场景

### 优点

- 能有效减少指标冗余。
- 适合处理多个相关变量。
- 既能做降维，也能做综合评价。
- 数学结构清晰，常用于论文中的综合指数构建。

### 局限

- 主成分是线性组合，解释性有时不如原指标直接。
- 结果受标准化方式影响较大。
- 若变量之间几乎不相关，PCA 的意义会下降。
- 主成分符号方向本身没有绝对意义，解释时需要统一方向。

### 适用场景

- 指标较多且彼此相关。
- 希望先压缩信息，再做排序或解释。
- 题目强调“综合因子”“降维”“提取主要信息”。

## 常见误区

### 1. 把 PCA 当成普通赋权法

PCA 更偏向变量变换和降维，不是直接给原指标配权重。

### 2. 不做标准化直接上 PCA

这样会导致量纲大的变量主导主成分。

### 3. 只看综合得分，不看载荷矩阵

如果不解释载荷矩阵，就浪费了 PCA 最大的结构分析价值。

## 一句话总结

PCA 的本质是“用少数几个新变量概括多数旧指标的信息”。它不只是一个排序工具，更是一个帮助你看清指标结构的降维工具。
