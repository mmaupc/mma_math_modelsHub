# 博弈论组合赋权

## 简介

博弈论组合赋权的任务，不是重新发明一套全新的权重，而是：

**把多套已有权重协调成一套更平衡的最终权重。**

最典型的场景是：

- 你已经有一套主观权重，例如 AHP；
- 你又有一套客观权重，例如熵权法、CRITIC 法；
- 但你不想只信其中一种。

这时就可以把不同权重方法看成不同“参与者”，让它们通过一种“协调博弈”的方式，形成最终组合权重。

所以它特别适合下面这类建模思路：

> 既想保留专家经验，又想保留数据信息。

---

## 它在解决什么问题

假设我们已经通过 $L$ 种方法得到了 $L$ 套权重向量：

$$
w^{(1)}, w^{(2)}, \dots, w^{(L)}.
$$

例如：

- $w^{(1)}$ 来自 AHP；
- $w^{(2)}$ 来自熵权法；
- $w^{(3)}$ 来自 CRITIC 法。

这时问题就变成了：

> 最终到底该用哪一套？

如果直接随便选一套，容易有片面性；
如果简单取平均，又等于默认“所有方法完全同等可信”。

博弈论组合赋权正是为了解决这个“如何整合多套权重”的问题。

---

## 数学原理

### 1. 组合权重的基本形式

设组合系数为 $\alpha_1,\alpha_2,\dots,\alpha_L$，则组合权重写成

$$
w = \sum_{k=1}^L \alpha_k w^{(k)},
$$

并满足

$$
\alpha_k \ge 0, \qquad \sum_{k=1}^L \alpha_k = 1.
$$

这里：

- $w^{(k)}$ 是第 $k$ 套基础权重；
- $\alpha_k$ 表示第 $k$ 套权重在最终组合中占多大比例。

### 2. 它为什么叫“博弈论”

这里的“博弈”不是指真的做动态博弈，而是指：

- 每套基础权重都代表一种立场；
- 最终组合权重要尽量兼顾各方；
- 不让某一套方法完全压制其他方法。

可以把它理解成一种“协调折中”的思想。

### 3. 一种常见的教材公式

在很多教材和论文里，常见的博弈论组合赋权写法是：

$$
\sum_{j=1}^L \alpha_j \, \bigl(w^{(i)}\bigr)^\top w^{(j)} = \bigl(w^{(i)}\bigr)^\top w^{(i)},
\qquad i=1,2,\dots,L.
$$

把它写成矩阵形式更清楚。定义

$$
G_{ij} = \bigl(w^{(i)}\bigr)^\top w^{(j)},
$$

$$
d_i = \bigl(w^{(i)}\bigr)^\top w^{(i)}.
$$

则有

$$
G\alpha = d.
$$

求出 $\alpha$ 后，再做非负化与归一化，就得到最终组合系数。

### 4. 这个公式在直觉上是什么意思

它本质上是在做一件事：

**让最终组合权重尽量同时靠近各套原始权重。**

因此：

- 如果某两套权重彼此很接近，它们在组合中通常更容易形成“共识”；
- 如果某一套权重和其他方法差异很大，它不一定被彻底否定，但往往会被自动下调影响力。

---

## 为什么不直接取平均

很多初学者会说：

> 既然有 3 套权重，那直接平均不就行了吗？

问题在于，简单平均隐含了一个很强的前提：

> 三套权重同样可靠、同样稳定、同样值得信任。

这在现实里往往并不成立。

例如：

- AHP 体现专家偏好；
- 熵权法体现数据离散性；
- CRITIC 体现区分度和指标冲突性。

它们提供的信息来源不同，彼此之间也可能并不一致。博弈论组合赋权的价值，就是用一个更有理论依据的方式做折中，而不是机械平均。

---

## 一个直觉例子

假设有 3 个指标、3 套基础权重：

$$
w^{(1)} = (0.60, 0.25, 0.15),
$$

$$
w^{(2)} = (0.30, 0.40, 0.30),
$$

$$
w^{(3)} = (0.28, 0.42, 0.30).
$$

从肉眼看就能发现：

- 第 2 套和第 3 套很接近；
- 第 1 套和另外两套差别更大。

这时如果做简单平均，三者地位完全相同；
但如果用博弈论组合赋权，最终结果通常会更偏向第 2 套和第 3 套的“共识区域”。

也就是说：

**谁更接近群体共识，谁在最终组合中往往更占优势。**

## 标准流程

博弈论组合赋权一般按下面 5 步进行：

### 第一步：先得到多套基础权重

例如：

- AHP；
- 熵权法；
- CRITIC；
- 变异系数法等。

### 第二步：建立组合模型

设

$$
w = \sum_{k=1}^L \alpha_k w^{(k)}.
$$

### 第三步：求组合系数 $\alpha_k$

常见做法就是求解前面的线性方程组

$$
G\alpha = d.
$$

### 第四步：做非负化与归一化

实际求解中，某些 $\alpha_k$ 可能出现负值。常见处理方式包括：

- 截断为 0 后重新归一化；
- 或改用带约束优化模型。

教学和竞赛中，前者更常见。

### 第五步：得到最终组合权重并进入后续评价

算出组合权重后，通常还要继续接：

- 加权求和；
- TOPSIS；
- VIKOR；
- 模糊综合评价等。

## 案例讲解：社区养老服务中心综合评价

### 问题背景

某区要对 4 个社区养老服务中心做综合评价。评价指标包括：

- 服务可达性；
- 医护保障；
- 活动丰富度；
- 老人满意度；
- 运营成本压力。

研究团队已经分别得到了 3 套权重：

- AHP 权重：体现专家经验；
- 熵权法权重：体现数据离散性；
- CRITIC 权重：体现指标对比强度与冲突性。

现在他们不想“只听专家的”，也不想“只听数据的”，于是采用博弈论组合赋权。

---

## 可运行 Python 代码

下面示例分两部分：

1. 先把 AHP、熵权法、CRITIC 三套权重组合起来；
2. 再用这套组合权重做一个简单的综合评分示例。

```python
import numpy as np
import pandas as pd


weight_dict = {
    "AHP": np.array([0.28, 0.24, 0.16, 0.22, 0.10]),
    "Entropy": np.array([0.20, 0.18, 0.24, 0.16, 0.22]),
    "CRITIC": np.array([0.18, 0.26, 0.20, 0.14, 0.22]),
}

# 逐行堆叠，每一行是一套基础权重
W = np.vstack(list(weight_dict.values()))

# 构造 G 和 d
G = W @ W.T
d = np.einsum("ij,ij->i", W, W)

# 求解组合系数
alpha = np.linalg.lstsq(G, d, rcond=None)[0]

# 简化教学处理：若出现负值，截断后重新归一化
alpha = np.clip(alpha, 0, None)
if np.isclose(alpha.sum(), 0):
    alpha = np.ones_like(alpha) / len(alpha)
else:
    alpha = alpha / alpha.sum()

# 组合权重
combined_weight = alpha @ W
combined_weight = combined_weight / combined_weight.sum()


data = pd.DataFrame(
    {
        "中心": ["E1", "E2", "E3", "E4"],
        "服务可达性": [88, 82, 90, 79],
        "医护保障": [84, 91, 80, 86],
        "活动丰富度": [76, 88, 81, 85],
        "老人满意度": [90, 87, 89, 84],
        "运营成本压力": [58, 62, 55, 60],
    }
)

indicator_types = {
    "服务可达性": "benefit",
    "医护保障": "benefit",
    "活动丰富度": "benefit",
    "老人满意度": "benefit",
    "运营成本压力": "cost",
}


def positive_transform(df, indicator_types):
    transformed = df.copy()
    for col, col_type in indicator_types.items():
        if col_type == "cost":
            transformed[col] = transformed[col].max() - transformed[col] + transformed[col].min()
    return transformed


def min_max_normalize(df):
    result = pd.DataFrame(index=df.index)
    for col in df.columns:
        col_min = df[col].min()
        col_max = df[col].max()
        if np.isclose(col_max, col_min):
            result[col] = 1.0
        else:
            result[col] = (df[col] - col_min) / (col_max - col_min)
    return result


X = positive_transform(data[list(indicator_types.keys())], indicator_types)
Z = min_max_normalize(X)
scores = Z.to_numpy() @ combined_weight

result = data[["中心"]].copy()
result["综合得分"] = scores
result = result.sort_values("综合得分", ascending=False).reset_index(drop=True)

print("基础权重：")
print(pd.DataFrame(W, index=list(weight_dict.keys()), columns=list(indicator_types.keys())).round(4))
print("\n组合系数 alpha：")
print(pd.Series(alpha, index=list(weight_dict.keys())).round(4))
print("\n组合权重：")
print(pd.Series(combined_weight, index=list(indicator_types.keys())).round(4))
print("\n排序结果：")
print(result.round(4))
```

## 结果应该怎么解释

### 1. 组合系数 $\alpha$

这一步告诉你：

- 哪套基础权重在最终组合中占比更高；
- 哪套权重与整体共识更接近。

### 2. 组合权重

这才是后续综合评价真正要用的权重。

它通常会同时保留：

- 主观偏好信息；
- 客观数据信息。

### 3. 如果某套权重占比很低

不代表它“完全错了”，更常见的解释是：

> 它和其他权重方法差异较大，因此在协调过程中影响被削弱了。

---

## 它和“主客观平均法”的区别

很多论文会写“主观权重和客观权重各占 50%”，这当然也可以，但那本质上是人为指定的。

博弈论组合赋权和它的区别在于：

- 平均法：先规定谁占多少；
- 博弈论组合：让模型根据多套权重的相互关系去协调。

因此它通常更有“方法论上的说服力”。

---

## 使用时的几个注意点

### 1. 基础权重本身必须靠谱

组合赋权并不能拯救质量很差的基础权重。它只是在“已有结果”之间做协调。

### 2. 负系数要解释

如果直接解方程组得到负系数，说明不同权重之间冲突较强。教学里常用“截断后归一化”，但在正式研究中最好说明处理理由。

### 3. 组合方法不止一种

博弈论组合赋权不是唯一的组合思想，常见的还有：

- 简单平均；
- 最小偏差法；
- 最小信息熵法等。

所以写论文时最好说明你采用的是哪种组合模型。

---

## 优缺点与适用场景

### 优点

- 能整合主观与客观信息；
- 比简单平均更有理论依据；
- 很适合用于多种赋权结果的协调；
- 能提高最终评价结果的说服力。

### 局限

- 仍然依赖基础权重本身的质量；
- 不同组合模型可能得到不同结果；
- 若权重之间差异很大，结果解释要更谨慎；
- 初学者容易只会套公式，而忽略“为什么要组合”。

### 适用场景

- 同时拥有多套权重结果；
- 既想体现专家经验，又想吸收客观数据；
- 论文中需要展示“主客观结合”的方法设计；
- 后续还要接 TOPSIS、VIKOR、FCE 等综合评价模型。

---

## 一句话总结

博弈论组合赋权的本质，不是重新造一套权重，而是让多套已有权重通过“协调折中”形成一个更平衡的最终版本。
