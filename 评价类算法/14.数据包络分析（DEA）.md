# 数据包络分析（DEA）

## 简介

数据包络分析（Data Envelopment Analysis, DEA）是一类专门做**效率评价**的方法。


它和前面那些“综合打分”方法最大的不同在于：

- 加权求和、TOPSIS、VIKOR 这些方法，通常是在比较“综合表现好不好”；
- DEA 比较的是“投入和产出之间的效率高不高”。

所以 DEA 最适合的问题，不是“谁总分更高”，而是：

> 在多投入、多产出的条件下，谁更会用资源？

这类场景非常多，比如：

- 医院科室效率评价；
- 银行网点运营效率评价；
- 学校、企业分部、物流中心的资源配置效率评价。

---

## 它在解决什么问题

假设我们要评价若干个对象，这些对象都做同一类事情，例如：

- 都是医院科室；
- 都是银行分支；
- 都是学校学院；
- 都是制造车间。

DEA 把每个评价对象称为一个**决策单元**（Decision Making Unit, DMU）。

对每个 DMU，我们通常都有两类指标：

- **投入指标**：用了多少资源；
- **产出指标**：做出了多少结果。

于是 DEA 想回答的问题就变成了：

> 在“投入”和“产出”都不止一个指标的情况下，怎样公平地比较这些对象谁更有效率？

这恰好是 DEA 的强项。

---

## 一个最直观的例子

先看一个只有 1 个投入、1 个产出的极简例子。

假设有两家门店：

| 门店 | 员工数 | 月销售额 |
| --- | ---: | ---: |
| A | 10 | 100 |
| B | 8 | 96 |

如果我们直接看销售额，A 更高；
但如果看“每个员工创造的销售额”：

$$
\text{A 的效率} = \frac{100}{10} = 10,
$$

$$
\text{B 的效率} = \frac{96}{8} = 12.
$$

这时反而是 B 更高效。

这说明：

**效率评价和规模评价不是一回事。**

DEA 所做的事情，就是把这种“产出 / 投入”的思想，推广到**多投入、多产出**的情形。

---

## 数学原理

### 1. DEA 的基本对象

设共有 $n$ 个 DMU。

对第 $k$ 个 DMU：

- 输入向量为

$$
x_k = (x_{k1}, x_{k2}, \dots, x_{km})^\top,
$$

- 输出向量为

$$
y_k = (y_{k1}, y_{k2}, \dots, y_{ks})^\top.
$$

其中：

- $m$ 是投入指标数；
- $s$ 是产出指标数。

### 2. 为什么 DEA 不需要先拍脑袋给权重

DEA 最早的直觉形式可以写成“加权产出比加权投入”：

$$
h_o = \frac{\sum_{r=1}^s u_r y_{or}}{\sum_{j=1}^m v_j x_{oj}},
$$

其中：

- $u_r$ 是产出权重；
- $v_j$ 是投入权重；
- $h_o$ 是第 $o$ 个 DMU 的效率值。

如果只看这个式子，你可能会担心：

> 那不是又要人为给权重了吗？

DEA 的妙处就在这里。它允许每个 DMU 选择**对自己最有利**的一组权重，但同时加上约束：

> 这组权重一旦选出来，对所有其他 DMU 算出来的效率都不能超过 1。

这样既保留了灵活性，又避免某个 DMU 通过极端权重“作弊”。

---

## CCR 模型：最经典的 DEA 模型

最经典的是 CCR 模型，它假设**规模报酬不变**（Constant Returns to Scale）。

教学中最常见的是**输入导向 CCR 包络模型**。对被评价的 DMU $o$，其形式为：

$$
\min_{\theta,\lambda} \ \theta
$$

满足

$$
\sum_{k=1}^n \lambda_k x_{kj} \le \theta x_{oj}, \qquad j=1,2,\dots,m,
$$

$$
\sum_{k=1}^n \lambda_k y_{kr} \ge y_{or}, \qquad r=1,2,\dots,s,
$$

$$
\lambda_k \ge 0, \qquad k=1,2,\dots,n.
$$

这里：

- $\theta$ 是效率值；
- $\lambda_k$ 是参考组合系数；
- $\theta$ 越小，说明在保持当前产出的条件下，理论上还能把投入压缩得更多；
- 最优值 $\theta^*$ 满足 $0 < \theta^* \le 1$。

### 这个模型到底在说什么

它实际上在问：

> 能不能用样本中其他 DMU 的某种线性组合，在不降低产出的前提下，以更少的投入“包住”当前 DMU？

如果能，就说明当前 DMU 还不够高效；
如果不能，就说明它已经站在效率前沿上。

---

## 为什么叫“包络”

从几何上看，DEA 会根据样本中表现较好的 DMU 构造出一个“效率前沿面”。

- 位于前沿面上的 DMU，效率值通常为 1；
- 位于前沿面下方的 DMU，效率值小于 1。

“包络”这个词，就是因为这个前沿面像一层壳，把所有 DMU 包在里面。

---

## 效率值应该怎么解释

在输入导向 CCR 模型里：

### 1. 若 $\theta^* = 1$

通常称该 DMU 在当前样本中**相对有效**。

但更严格地说，如果还想判断“强有效”，通常还要继续检查松弛变量是否为 0。入门阶段先记住：

- $\theta^*=1$：已经处在样本前沿附近；
- $\theta^*<1$：存在改进空间。

### 2. 若 $\theta^* < 1$

说明该 DMU 可以在**不减少产出**的前提下，把全部投入按比例缩减到原来的 $\theta^*$ 倍。

例如：

$$
\theta^* = 0.85
$$

表示理论上可把投入整体压缩到原来的 $85\%$。

---

## CCR 和 BCC 的区别

DEA 入门时最容易见到的两个模型是：

### CCR 模型

- 假设规模报酬不变；
- 更适合分析“综合技术效率”。

### BCC 模型

- 允许规模报酬可变；
- 在 CCR 约束基础上增加

$$
\sum_{k=1}^n \lambda_k = 1.
$$

- 更适合把“纯技术效率”和“规模效率”区分开来。

如果你刚开始学 DEA，通常先把 CCR 理清楚，再看 BCC 会更顺。

---

## 一个能手算的直觉例子

假设有 3 个 DMU，只有 1 个投入和 1 个产出：

| DMU | 投入 | 产出 |
| --- | ---: | ---: |
| A | 10 | 10 |
| B | 8 | 10 |
| C | 6 | 7 |

从最简单的“产出 / 投入”看：

$$
\text{A} = 1.00,\qquad \text{B} = 1.25,\qquad \text{C} \approx 1.17.
$$

显然 B 最有效率。

如果把这件事放到 DEA 的语言里：

- B 站在效率前沿上；
- A 和 C 都可能被 B“包络”；
- A 如果想达到和 B 一样的效率，就得减少投入或者提高产出。

在多投入、多产出的场景中，DEA 就是在做这个“找效率前沿”的过程，只不过由简单比值变成了线性规划。

---

## 标准流程

DEA 建模通常按下面 6 步进行：

### 第一步：确定 DMU

各 DMU 必须具有可比性，也就是“做的是同一类事情”。

### 第二步：划分投入和产出指标

这是 DEA 最关键的建模前提。

- 投入是资源消耗；
- 产出是成果表现。

### 第三步：选择模型

常见选择包括：

- CCR 或 BCC；
- 输入导向或输出导向。

### 第四步：逐个 DMU 求解线性规划

对每个 DMU 单独求一次效率值。

### 第五步：识别效率前沿与参考对象

效率值为 1 的 DMU 通常处于前沿上；
效率值小于 1 的 DMU 可以找到其对标参考组合。

### 第六步：给出管理解释

DEA 的真正价值不只是“谁效率高”，还包括：

- 谁是标杆；
- 谁可以向谁学习；
- 哪些 DMU 的投入还有压缩空间。

---

## 案例讲解：医院科室运营效率评价

### 问题背景

某医院想比较 5 个科室的运营效率。投入指标包括：

- 医护人数；
- 床位数；
- 年度经费。

产出指标包括：

- 年出院人数；
- 患者满意度。

这个问题非常适合 DEA，因为这里同时存在：

- 多个投入；
- 多个产出；
- 并且评价重点是“资源使用效率”，不是单纯拼规模。

---

## 可运行 Python 代码

下面示例实现的是**输入导向 CCR 包络模型**。代码会输出：

- 每个科室的 CCR 效率值；
- 每个科室的参考同类（peer）；
- 哪些科室处于效率前沿。

```python
import numpy as np
import pandas as pd
from scipy.optimize import linprog


data = pd.DataFrame(
    {
        "科室": ["内科", "外科", "儿科", "骨科", "急诊"],
        "医护人数": [56, 63, 41, 47, 58],
        "床位数": [92, 105, 68, 74, 80],
        "年度经费": [12.5, 15.2, 9.8, 10.6, 13.4],
        "出院人数": [4200, 4680, 3550, 3890, 4410],
        "满意度": [92, 89, 95, 91, 90],
    }
)

inputs = data[["医护人数", "床位数", "年度经费"]].to_numpy(dtype=float)
outputs = data[["出院人数", "满意度"]].to_numpy(dtype=float)
names = data["科室"].tolist()


def dea_ccr_input(X, Y, names, tol=1e-6):
    n, m = X.shape
    s = Y.shape[1]
    rows = []

    for o in range(n):
        # 决策变量: [theta, lambda_1, ..., lambda_n]
        c = np.zeros(n + 1)
        c[0] = 1.0

        A_ub = []
        b_ub = []

        # 输入约束:
        # sum(lambda_k * x_kj) - theta * x_oj <= 0
        for j in range(m):
            row = np.zeros(n + 1)
            row[0] = -X[o, j]
            row[1:] = X[:, j]
            A_ub.append(row)
            b_ub.append(0.0)

        # 输出约束:
        # -sum(lambda_k * y_kr) <= -y_or
        for r in range(s):
            row = np.zeros(n + 1)
            row[1:] = -Y[:, r]
            A_ub.append(row)
            b_ub.append(-Y[o, r])

        bounds = [(0, 1)] + [(0, None)] * n

        res = linprog(
            c,
            A_ub=np.array(A_ub, dtype=float),
            b_ub=np.array(b_ub, dtype=float),
            bounds=bounds,
            method="highs",
        )

        if not res.success:
            rows.append({"科室": names[o], "CCR效率值": np.nan, "参考同类": "求解失败"})
            continue

        theta = res.x[0]
        lambdas = res.x[1:]
        peers = [names[k] for k, lam in enumerate(lambdas) if lam > tol]

        rows.append(
            {
                "科室": names[o],
                "CCR效率值": theta,
                "参考同类": "、".join(peers) if peers else "无",
            }
        )

    return pd.DataFrame(rows)


result = dea_ccr_input(inputs, outputs, names)

print("DEA 评价结果：")
print(result.sort_values("CCR效率值", ascending=False).reset_index(drop=True).round(4))

efficient_units = result.loc[np.isclose(result["CCR效率值"], 1.0, atol=1e-4), "科室"].tolist()
print("\n处于效率前沿的科室：")
print(efficient_units)
```

---

## 结果应该怎么解释

### 1. CCR 效率值

- 等于 1：说明该科室在当前样本中处于效率前沿；
- 小于 1：说明还有投入压缩空间。

### 2. 参考同类

如果某个科室效率不足，DEA 会给出一个“参考组合”，告诉你它更接近哪些高效科室的线性组合。

这比单纯说“你效率低”更有管理意义，因为它还能回答：

> 你应该向谁学？

### 3. 相对有效不等于绝对最好

DEA 的结论依赖样本集合。某个科室效率值等于 1，只说明它在**这批样本**里已经站在前沿上，并不等于它没有任何提升空间。

---

## 使用 DEA 时最容易踩的坑

### 1. DMU 必须同质

如果把完全不同类型的对象放在一起比较，例如把三甲医院和社区诊所混在一起，DEA 结果往往没有意义。

### 2. 投入和产出不能乱放

DEA 的建模质量，很大程度上取决于：

- 哪些变量被视为投入；
- 哪些变量被视为产出。

如果方向放反了，结论会失真。

### 3. 样本数不能太少

如果 DMU 太少、指标太多，很多对象都会轻易落在效率前沿上，导致区分力不足。

### 4. DEA 更适合效率，不适合偏好

如果你的问题重点是“谁更值得选”，而不是“谁更高效”，那 DEA 就不一定是最合适的工具。

---

## 优缺点与适用场景

### 优点

- 能同时处理多投入、多产出；
- 不需要预先人为指定指标权重；
- 很适合做效率前沿分析；
- 不仅给出效率值，还能提供参考对象。

### 局限

- 只适合可比的同类 DMU；
- 对样本数量比较敏感；
- 对异常值较敏感；
- 评价的是相对效率，不是绝对最优。

### 适用场景

- 医院、学校、银行、企业分支机构效率评价；
- 有明确投入和产出结构的问题；
- 重点在资源配置效率而不是偏好排序的问题。

---

## 一句话总结

DEA 的核心不是看谁规模大，而是看谁能用更少的投入做出更多的产出。
