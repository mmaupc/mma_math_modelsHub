# ELECTRE法

## 简介


ELECTRE 是一类非常经典的 outranking 方法，中文通常翻译为：

- 超越关系法；
- 级别高于关系法；
- 优势关系法。

它的核心思想是：

**一个方案要想“超越”另一个方案，不仅要有足够多的支持证据，还不能在关键指标上输得太惨。**

这和简单加权总分法非常不一样。加权总分法允许“这里差一点、那里补回来”；而 ELECTRE 更强调：

> 有些短板不是靠别的优势就能轻易补偿掉的。


因此，ELECTRE 特别适合存在“底线约束”或“否决思维”的问题。

---

## 它在解决什么问题

假设有 $m$ 个方案、$n$ 个指标，评价矩阵为

$$
X = (x_{ij})_{m \times n}.
$$

ELECTRE 想回答的问题不是：

> 谁的总分最高？

而是：

> 方案 $a$ 有没有足够理由被认为“整体上优于”方案 $b$？

这个判断通常包含两部分：

1. 多数高权重指标是否支持 $a$ 不差于 $b$；
2. 是否存在某个关键指标，虽然多数票支持 $a$，但它在这一项上差得太离谱，以至于不能接受。

这就是 ELECTRE 的“一致性 + 不一致性”逻辑。

---

## 这篇文档采用哪个版本

ELECTRE 其实是一个方法家族，常见版本包括：

- ELECTRE I；
- ELECTRE II；
- ELECTRE III；
- ELECTRE TRI 等。

不同版本在阈值设置、排序方式、否决机制上都不完全一样。

为了适合初学者入门，这篇文档主要介绍**ELECTRE I 的简化教学版**。它足够体现 ELECTRE 的核心思想，也最适合先建立直觉。

---

## 数学原理

### 1. 指标同向化与标准化

通常先把所有指标处理成“越大越好”，再做标准化，得到矩阵

$$
Z = (z_{ij})_{m \times n}.
$$

这样不同指标就可以在同一框架里比较。

### 2. 一致集与不一致集

对任意两个方案 $a$ 和 $b$，定义：

一致集

$$
J^+(a,b)=\{j \mid z_j(a)\ge z_j(b)\},
$$

不一致集

$$
J^-(a,b)=\{j \mid z_j(a)<z_j(b)\}.
$$

也就是说：

- 在一致集里的指标，支持“$a$ 不差于 $b$”；
- 在不一致集里的指标，支持“$b$ 优于 $a$”。

### 3. 一致度

设各指标权重为 $w_j$，满足

$$
w_j \ge 0, \qquad \sum_{j=1}^n w_j = 1.
$$

则 $a$ 相对 $b$ 的一致度定义为

$$
C(a,b) = \sum_{j \in J^+(a,b)} w_j.
$$

$C(a,b)$ 越大，说明支持 $a$ 不差于 $b$ 的权重总和越大。

你可以把它理解成：

**支持 $a$ 超越 $b$ 的“票数权重”有多大。**

### 4. 不一致度

仅有多数支持还不够，因为可能存在某个关键指标，$a$ 在那一项上输得太厉害。

于是 ELECTRE I 的简化教学版常定义不一致度为

$$
D(a,b) = \max_{j \in J^-(a,b)} \frac{|z_j(a)-z_j(b)|}{R_j},
$$

其中

$$
R_j = \max_i z_{ij} - \min_i z_{ij}
$$

表示第 $j$ 个指标的波动范围。

如果 $J^-(a,b)$ 为空，即 $a$ 在所有指标上都不差于 $b$，则通常令

$$
D(a,b)=0.
$$

$D(a,b)$ 越大，说明在那些不支持 $a$ 的指标里，至少有一项差得很明显。

### 5. 超越关系判定

设一致性阈值为 $c^*$，不一致性阈值为 $d^*$。如果满足

$$
C(a,b)\ge c^*, \qquad D(a,b)\le d^*,
$$

就认为：

$$
a \; S \; b,
$$

即“方案 $a$ 超越方案 $b$”。

这个判定很像现实中的决策逻辑：

- 支持票要够多；
- 反对证据不能太强。

---

## 为什么 ELECTRE 很像“带否决权的投票”

ELECTRE 的思想可以直观理解成两句话：

### 1. 多数支持

如果很多重要指标都支持 $a$ 不差于 $b$，那 $a$ 确实应该有机会超越 $b$。

### 2. 关键指标不能输得太惨

如果 $a$ 在某个关键指标上明显落后，那么即使多数指标支持它，也可能不能认定它超越 $b$。

这就是 ELECTRE 和加权求和法最大的区别：

**ELECTRE 允许“有底线”，而不是一切都能补偿。**

---

## 一个能手算的直觉例子

假设我们比较两个方案 A 和 B，指标权重如下：

| 指标 | 权重 |
| --- | ---: |
| 收益 | 0.25 |
| 成本 | 0.20 |
| 安全性 | 0.35 |
| 技术成熟度 | 0.20 |

现在假设：

- A 在收益、成本、技术成熟度上都优于 B；
- 但 A 在安全性上明显劣于 B。

那么 A 相对 B 的一致度可能是

$$
C(A,B)=0.25+0.20+0.20=0.65.
$$

看起来支持票已经不少了。

但如果安全性这个指标上的差距特别大，不一致度可能会很高，例如

$$
D(A,B)=0.92.
$$

如果阈值设为

$$
c^*=0.60, \qquad d^*=0.60,
$$

那么虽然

$$
C(A,B)=0.65 \ge 0.60,
$$

但因为

$$
D(A,B)=0.92 > 0.60,
$$

最终仍然不能认定 A 超越 B。

这个例子很好地说明了 ELECTRE 的精神：

**多数支持不够，关键短板也必须过关。**

## 标准流程

ELECTRE I 常按下面 6 步进行：

### 第一步：统一指标方向

把所有指标转成“越大越好”。

### 第二步：标准化

把不同量纲的指标拉到可比较尺度。

### 第三步：计算每一对方案的一致度和不一致度

得到一致度矩阵 $C$ 和不一致度矩阵 $D$。

### 第四步：设定阈值

确定

$$
c^*, \qquad d^*.
$$

教学示例中，常用所有非对角元素的均值作为经验阈值；真实问题里则更建议结合专家判断设定。

### 第五步：构造超越关系矩阵

判断每一对方案是否满足

$$
C(a,b)\ge c^*, \qquad D(a,b)\le d^*.
$$

### 第六步：做筛选或辅助排序

ELECTRE 的结果通常是：

- 一个超越关系图；
- 或一组“核心候选方案”；
- 有时再配合净超越数做辅助排序。

## 案例讲解：垃圾处理技术路线筛选

### 问题背景

某市准备在 4 种垃圾处理技术路线中进行筛选，候选方案包括：

- A：焚烧发电；
- B：生物处理；
- C：热解气化；
- D：卫生填埋升级。

评价指标包括：

- 资源化收益：越大越好；
- 建设成本：越小越好；
- 排放风险：越小越好；
- 占地强度：越小越好；
- 技术成熟度：越大越好。

这个问题很适合 ELECTRE，因为决策层明确有“底线思维”：

> 不能因为收益高，就容忍排放风险过大。

也就是说，这不是一个单纯靠总分就能解决的问题。

## 可运行 Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "方案": ["A", "B", "C", "D"],
        "资源化收益": [88, 74, 85, 58],
        "建设成本": [62, 48, 70, 40],
        "排放风险": [36, 24, 33, 45],
        "占地强度": [42, 30, 38, 55],
        "技术成熟度": [92, 80, 76, 88],
    }
)

weights = pd.Series(
    {
        "资源化收益": 0.26,
        "建设成本": 0.20,
        "排放风险": 0.24,
        "占地强度": 0.12,
        "技术成熟度": 0.18,
    }
)
weights = weights / weights.sum()

indicator_types = {
    "资源化收益": "benefit",
    "建设成本": "cost",
    "排放风险": "cost",
    "占地强度": "cost",
    "技术成熟度": "benefit",
}


def positive_transform(df, indicator_types):
    transformed = df.copy()
    for col, col_type in indicator_types.items():
        if col_type == "cost":
            transformed[col] = transformed[col].max() - transformed[col] + transformed[col].min()
    return transformed


def min_max_normalize(df):
    result = pd.DataFrame(index=df.index)
    for col in df.columns:
        col_min = df[col].min()
        col_max = df[col].max()
        if np.isclose(col_max, col_min):
            result[col] = 1.0
        else:
            result[col] = (df[col] - col_min) / (col_max - col_min)
    return result


X = positive_transform(data[list(indicator_types.keys())], indicator_types)
Z = min_max_normalize(X)
ranges = Z.max(axis=0) - Z.min(axis=0)

names = data["方案"].tolist()
m = len(names)
concordance = pd.DataFrame(np.zeros((m, m)), index=names, columns=names)
discordance = pd.DataFrame(np.zeros((m, m)), index=names, columns=names)

for i in range(m):
    for j in range(m):
        if i == j:
            continue

        diff = Z.iloc[i] - Z.iloc[j]
        support = diff >= 0
        oppose = diff < 0

        concordance.iloc[i, j] = weights[support.index[support]].sum()

        if oppose.any():
            discordance.iloc[i, j] = (
                diff[oppose].abs() / ranges[oppose.index[oppose]].replace(0, 1)
            ).max()
        else:
            discordance.iloc[i, j] = 0.0

# 教学示例里，用非对角元素均值作为经验阈值
mask = ~np.eye(m, dtype=bool)
c_star = concordance.to_numpy()[mask].mean()
d_star = discordance.to_numpy()[mask].mean()

outranking = ((concordance >= c_star) & (discordance <= d_star)).astype(int)
np.fill_diagonal(outranking.values, 0)

# 没有被任何其他方案超越的方案，作为一个简化版“核心候选集”
kernel = outranking.index[(outranking.sum(axis=0) == 0)].tolist()

# 净超越数只作为辅助排序，不是 ELECTRE 的唯一正式输出
net = outranking.sum(axis=1) - outranking.sum(axis=0)
result = pd.DataFrame(
    {
        "方案": names,
        "净超越数": net,
    }
).sort_values("净超越数", ascending=False).reset_index(drop=True)

print("正向化后的数据：")
print(X.round(4))
print("\n标准化矩阵：")
print(Z.round(4))
print("\n一致度矩阵：")
print(concordance.round(4))
print("\n不一致度矩阵：")
print(discordance.round(4))
print(f"\n阈值: c*={c_star:.4f}, d*={d_star:.4f}")
print("\n超越关系矩阵：")
print(outranking)
print(f"\n核心候选集: {kernel}")
print("\n按净超越数给出的辅助排序：")
print(result)
```

---

## 结果应该怎么解释

### 1. 一致度矩阵

它表示“支持方案 $a$ 不差于方案 $b$ 的权重总和”。

### 2. 不一致度矩阵

它表示“在反对 $a$ 超越 $b$ 的那些指标里，最严重的反对程度有多强”。

### 3. 超越关系矩阵

如果矩阵中某个位置为 1，表示对应的超越关系成立。

这时 ELECTRE 给出的不再是简单总分，而是一个“谁足以压过谁”的关系网络。

### 4. 核心候选集

如果某个方案没有被其他方案超越，它可以被视为较强的候选方案。

这比“只给一个总分最高者”更符合很多带风险底线的问题。

### 5. 净超越数

这只是一个辅助阅读结果的方法。它能帮助我们快速观察哪些方案在超越关系网络里更占优势，但不应替代 ELECTRE 的核心逻辑。

---

## ELECTRE 和 PROMETHEE 的区别

这两个方法都做两两比较，但气质不同。

### PROMETHEE 更像连续打分

它通过偏好函数和净偏好流，比较自然地给出全排序。

### ELECTRE 更像带否决权的筛选

它强调：

- 多数支持够不够；
- 有没有关键短板直接把方案卡掉。

因此：

- 想做完整排序，PROMETHEE 通常更顺手；
- 想做“带底线约束”的筛选，ELECTRE 更贴近实际管理逻辑。

---

## 优缺点与适用场景

### 优点

- 能体现底线思维和否决逻辑；
- 不容易让严重短板被其他指标补偿掉；
- 很适合筛选而不是单纯打分；
- 更贴近很多真实管理场景。

### 局限

- 阈值设置较敏感；
- 结果常常是部分排序，不一定给唯一第一名；
- 初学者理解成本相对较高；
- ELECTRE 家族版本很多，实际使用时必须说明采用的是哪一种。

### 适用场景

- 高风险、高约束决策；
- 存在明显底线要求或否决项；
- 想做方案筛选而不只是简单总分排序；
- 环境治理、技术路线选择、政策方案比选等问题。

---

## 一句话总结

ELECTRE 的核心，就是只有当“支持证据足够多，而且反对证据不致命”时，一个方案才算真正超越另一个方案。
