# 优劣解距离法（TOPSIS）

TOPSIS 是评价类模型中非常经典的一种排序方法，全称是 Technique for Order Preference by Similarity to Ideal Solution。名字看起来很长，但思想其实非常直观：

一个好方案，应该尽量接近“理想最优方案”，同时尽量远离“理想最差方案”。

所以，TOPSIS 的核心不是单纯算加权和，而是看每个方案到“最优样板”和“最差样板”的距离。

## 这个方法在解决什么问题

假设有 $m$ 个待评价对象、$n$ 个评价指标。经过正向化、标准化和加权后，得到加权矩阵

$$
V = (v_{ij})_{m \times n}
$$

然后构造两个虚拟方案：

- 正理想解 $A^+$：每个指标都取最优值。
- 负理想解 $A^-$：每个指标都取最差值。

记第 $i$ 个方案到正理想解、负理想解的欧氏距离分别为

$$
D_i^+ = \sqrt{\sum_{j=1}^n (v_{ij} - v_j^+)^2}
$$

$$
D_i^- = \sqrt{\sum_{j=1}^n (v_{ij} - v_j^-)^2}
$$

则贴近度定义为

$$
C_i = \frac{D_i^-}{D_i^+ + D_i^-}
$$

其中 $0 \le C_i \le 1$。通常认为 $C_i$ 越大，方案越优。

## 数学原理

### 1. 先把所有指标统一成“越大越好”

TOPSIS 和加权求和法一样，不能直接处理方向混乱的指标。比如：

- 续航里程越大越好。
- 购置成本越小越好。
- 充电时间越小越好。

所以第一步仍然要做正向化。成本型指标常见处理方式之一是

$$
x'_{ij} = \max_i x_{ij} - x_{ij}
$$

或者写成归一化形式

$$
x'_{ij} = \frac{\max_i x_{ij} - x_{ij}}{\max_i x_{ij} - \min_i x_{ij}}
$$

### 2. 再做标准化

TOPSIS 最常见的是向量标准化：

$$
r_{ij} = \frac{x'_{ij}}{\sqrt{\sum_{i=1}^m (x'_{ij})^2}}
$$

这样做的目的是让不同指标处于可比较尺度，同时保留方案间的相对结构。

### 3. 引入权重

若第 $j$ 个指标权重为 $w_j$，则加权标准化矩阵为

$$
v_{ij} = w_j r_{ij}
$$

### 4. 正理想解和负理想解不是“真实方案”

正理想解定义为

$$
A^+ = (v_1^+, v_2^+, \dots, v_n^+)
$$

其中

$$
v_j^+ = \max_i v_{ij}
$$

负理想解定义为

$$
A^- = (v_1^-, v_2^-, \dots, v_n^-)
$$

其中

$$
v_j^- = \min_i v_{ij}
$$

请注意，这两个“理想解”通常只是拼出来的虚拟样板，并不一定真的存在于候选方案中。

### 5. 为什么贴近度这样定义

如果一个方案：

- 离正理想解很近，也就是 $D_i^+$ 很小；
- 离负理想解很远，也就是 $D_i^-$ 很大；

那么它的贴近度

$$
C_i = \frac{D_i^-}{D_i^+ + D_i^-}
$$

就会很大，自然更值得推荐。

这其实是一种非常符合直觉的排序逻辑。

## 一个直觉 example

假设你要选一辆车，只看三个指标：

- 续航越大越好。
- 价格越低越好。
- 充电越快越好。

此时你脑海里会自然形成两个参照物：

- 一辆“梦中情车”：续航最长、价格最低、充电最快。
- 一辆“最不想买的车”：续航最短、价格最高、充电最慢。

TOPSIS 做的，就是看每辆真实车辆更像哪一个。

所以它和加权求和法最大的不同在于：它不是只看“累计得了多少分”，而是看“你离最好有多近，离最差有多远”。

## 标准流程

### 第一步：指标正向化

把所有指标都转化为“越大越好”的形式。

### 第二步：标准化

常用向量标准化，得到矩阵 $R = (r_{ij})$。

### 第三步：加权

得到加权标准化矩阵 $V = (v_{ij})$。

### 第四步：构造正、负理想解

按列取最大值得到正理想解，按列取最小值得到负理想解。

### 第五步：计算距离

分别求每个方案到正理想解和负理想解的欧氏距离。

### 第六步：计算贴近度并排序

按 $C_i$ 从大到小排序即可。

## 建模案例：新能源公交车型选择

### 问题背景

某市准备采购一批新能源公交车，现有 5 款候选车型。评估指标如下：

- 续航里程：越大越好。
- 购置成本：越小越好。
- 百公里电耗：越小越好。
- 载客量：越大越好。
- 充电时间：越小越好。

决策层给出的权重为

$$
w = (0.25,\ 0.20,\ 0.20,\ 0.15,\ 0.20)
$$

现在希望利用 TOPSIS 对 5 款车型排序。

### 建模思路

这个案例非常适合 TOPSIS，因为：

- 有明显的多指标冲突，比如续航和成本往往不能同时最优。
- 采购决策天然会拿“理想车型”做参照。
- 希望看到每个方案距离理想方案到底有多远。

### Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "车型": ["M1", "M2", "M3", "M4", "M5"],
        "续航里程": [420, 380, 460, 400, 440],
        "购置成本": [95, 88, 102, 90, 97],
        "百公里电耗": [78, 72, 80, 68, 74],
        "载客量": [78, 72, 80, 76, 75],
        "充电时间": [1.6, 1.2, 1.8, 1.0, 1.4],
    }
)

weights = pd.Series(
    {
        "续航里程": 0.25,
        "购置成本": 0.20,
        "百公里电耗": 0.20,
        "载客量": 0.15,
        "充电时间": 0.20,
    }
)

indicator_types = {
    "续航里程": "benefit",
    "购置成本": "cost",
    "百公里电耗": "cost",
    "载客量": "benefit",
    "充电时间": "cost",
}


def positive_transform(series, indicator_type):
    series = series.astype(float)
    max_val = series.max()
    min_val = series.min()

    if np.isclose(max_val, min_val):
        return pd.Series(np.ones(len(series)), index=series.index)

    if indicator_type == "benefit":
        return series
    if indicator_type == "cost":
        return max_val - series

    raise ValueError("未知指标类型")


processed = pd.DataFrame(index=data.index)
for col, indicator_type in indicator_types.items():
    processed[col] = positive_transform(data[col], indicator_type)

norm = processed / np.sqrt((processed**2).sum(axis=0))
weighted = norm.mul(weights, axis=1)

ideal_best = weighted.max(axis=0)
ideal_worst = weighted.min(axis=0)

d_pos = np.sqrt(((weighted - ideal_best) ** 2).sum(axis=1))
d_neg = np.sqrt(((weighted - ideal_worst) ** 2).sum(axis=1))
closeness = d_neg / (d_pos + d_neg)

result = data[["车型"]].copy()
result["D_plus"] = d_pos
result["D_minus"] = d_neg
result["贴近度"] = closeness
result = result.sort_values("贴近度", ascending=False).reset_index(drop=True)

print("加权标准化矩阵：")
print(weighted.round(4))

print("\n正理想解：")
print(ideal_best.round(4))

print("\n负理想解：")
print(ideal_worst.round(4))

print("\nTOPSIS 排序结果：")
print(result.round(4))
```

### 结果解读

运行后建议重点看三部分：

- 正理想解和负理想解。
- 每个方案的 $D_i^+$ 与 $D_i^-$。
- 最终贴近度 $C_i$。

如果某个车型：

- 到正理想解距离很小；
- 到负理想解距离很大；

那么它的贴近度就会更大，排序也会更靠前。

这意味着 TOPSIS 不一定偏爱“某单项极强”的方案，而往往更偏爱“整体离理想状态更近”的方案。

## TOPSIS 的优缺点与适用场景

### 优点

- 思路直观，容易解释给非数学背景读者。
- 同时考虑了“接近最优”和“远离最差”。
- 可以和 AHP、熵权法、CRITIC 等权重方法结合。
- 在方案排序问题中非常常见，适合论文写作。

### 局限

- 对标准化方式有一定敏感性。
- 正理想解是虚拟构造的，不一定真实存在。
- 默认指标之间相互可补偿。
- 若数据存在极端值，距离计算可能受到影响。

### 适用场景

- 多方案、多指标排序问题。
- 希望从“理想参照物”的角度理解方案优劣。
- 权重已知或可通过其他方法先求得。

## 常见误区

### 1. 忘记先做正向化

如果成本型指标没先处理，TOPSIS 的理想解就会被构造错。

### 2. 把正理想解当成真实方案

正理想解通常只是每一列最优值拼出来的“虚拟对象”，不一定真的存在。

### 3. 只看贴近度，不看数据背景

TOPSIS 给出的是排序工具，不会自动告诉你“为什么这个方案更适合现实约束”，解释时还要结合业务含义。

## 一句话总结

TOPSIS 的本质是“选那个最像理想方案、最不像最差方案的对象”。它很适合做多指标排序，因为它把“综合优”这件事转成了一个很有直觉的距离问题。
