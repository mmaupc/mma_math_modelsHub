# VIKOR法

## 简介

VIKOR 是一种经典的多指标决策方法，完整名称通常写作：

$$
\text{VIseKriterijumska Optimizacija I Kompromisno Resenje}
$$

它最核心的关键词是**妥协解（compromise solution）**。很多综合评价问题并不存在“所有指标都第一”的完美方案。常见情况往往是：

- 某个方案整体表现很好，但某一项关键指标很差；
- 某个方案没有特别亮眼，但几乎没有明显短板。

这时如果只看“总分”，容易忽略短板；如果只看“最差项”，又会忽略整体表现。VIKOR 就是在这两者之间找平衡。

---

## 它在解决什么问题

假设有 $m$ 个方案、$n$ 个指标，评价矩阵为

$$
F = (f_{ij})_{m \times n},
$$

其中 $f_{ij}$ 表示方案 $i$ 在指标 $j$ 上的表现。

VIKOR 想解决的是下面这类问题：

> 当没有任何一个方案在所有指标上都完美时，怎样找到一个“整体不错，且关键短板不至于太差”的折中方案？

因此，VIKOR 不只是简单排序，它更强调：

- **群体效益**：整体上离理想方案近不近；
- **个体遗憾**：最差那一项有没有特别糟。

---

## 数学原理

### 1. 先定义每个指标上的最好值和最坏值

为了书写统一，通常先把所有指标处理成“越大越好”的方向。然后对第 $j$ 个指标定义：

$$
f_j^* = \max_i f_{ij}, \qquad f_j^- = \min_i f_{ij}.
$$

其中：

- $f_j^*$ 是第 $j$ 个指标上的理想最好值；
- $f_j^-$ 是第 $j$ 个指标上的最差值。

### 2. 定义“离理想状态还有多远”

对方案 $i$、指标 $j$，定义归一化差距：

$$
d_{ij} = \frac{f_j^* - f_{ij}}{f_j^* - f_j^-}.
$$

如果某个指标对所有方案都一样，即 $f_j^* = f_j^-$，通常令该列差距为 $0$，因为它不提供区分信息。

$d_{ij}$ 的含义非常直观：

- $d_{ij} = 0$，说明方案 $i$ 在指标 $j$ 上已经达到当前样本中的最好水平；
- $d_{ij} = 1$，说明它在这个指标上处于当前样本中的最差水平；
- 值越小越好，表示离理想值越近。

### 3. 群体效益指标 $S_i$

VIKOR 用

$$
S_i = \sum_{j=1}^n w_j d_{ij}
$$

来衡量方案 $i$ 的总体损失，其中 $w_j$ 是第 $j$ 个指标的权重，且满足

$$
w_j \ge 0, \qquad \sum_{j=1}^n w_j = 1.
$$

$S_i$ 越小，说明这个方案在整体上越接近理想状态。

### 4. 个体遗憾指标 $R_i$

只看 $S_i$ 还不够，因为总损失小，并不代表没有明显短板。

于是 VIKOR 再定义

$$
R_i = \max_j \left( w_j d_{ij} \right).
$$

$R_i$ 表示方案 $i$ 在“最吃亏”的那个指标上，损失有多大。

所以：

- $S_i$ 看整体；
- $R_i$ 看最坏项。

### 5. 妥协排序指标 $Q_i$

为了把“整体表现”和“最差短板”综合起来，VIKOR 定义：

$$
Q_i = v \frac{S_i - S^*}{S^- - S^*} + (1-v)\frac{R_i - R^*}{R^- - R^*},
$$

其中：

$$
S^* = \min_i S_i, \qquad S^- = \max_i S_i,
$$

$$
R^* = \min_i R_i, \qquad R^- = \max_i R_i.
$$

$v \in [0,1]$ 是权衡参数，常取

$$
v = 0.5.
$$

它的含义是：

- $v$ 越大，越强调群体效益，也就是更看重整体表现；
- $v$ 越小，越强调个体遗憾，也就是更在意最差短板。

最终：

$$
Q_i \text{ 越小，方案越优。}
$$

---

## 为什么 VIKOR 不只是“另一种总分法”

如果你只做加权求和，本质上是在算平均意义上的好坏。

但现实中常常有这种情况：

- 方案 A：整体平均水平高，但在某个关键指标上明显偏弱；
- 方案 B：整体不如 A 那么亮眼，但各项都比较均衡。

很多决策者并不希望“平均分高”就自动掩盖掉关键短板。VIKOR 正是为了解决这个问题。

---

## 一个能手算的直觉例子

假设有两个方案，且已经把指标差距算成了“越小越好”的加权损失：

| 方案 | 指标 1 损失 | 指标 2 损失 |
| --- | ---: | ---: |
| A | 0.00 | 0.20 |
| B | 0.10 | 0.10 |

那么：

$$
S_A = 0.00 + 0.20 = 0.20, \qquad S_B = 0.10 + 0.10 = 0.20.
$$

也就是说，从总损失看，A 和 B 一样。

但如果看最大遗憾：

$$
R_A = \max(0.00, 0.20) = 0.20,
$$

$$
R_B = \max(0.10, 0.10) = 0.10.
$$

这就说明：

- A 的问题在于有一个比较明显的短板；
- B 虽然没有哪一项特别突出，但也没有明显短板。

如果你的决策更强调“不要有明显短板”，那么 B 会更受欢迎。这就是 VIKOR 的直觉核心。

---

## 妥协解判定条件

VIKOR 不只是给出一个 $Q$ 排名，还会进一步判断“第一名是否真的足够合理”。设按 $Q$ 从小到大排序后的前两名分别为 $a^{(1)}$ 和 $a^{(2)}$。

### 1. 可接受优势条件

要求

$$
Q(a^{(2)}) - Q(a^{(1)}) \ge DQ,
$$

其中

$$
DQ = \frac{1}{m-1}.
$$

这表示第一名要和第二名拉开足够差距。

### 2. 可接受稳定性条件

第一名 $a^{(1)}$ 还应该在 $S$ 排名或 $R$ 排名中至少有一个也是第一。

也就是说，它不能只是 $Q$ 上侥幸领先，而应该在“整体表现”或“最大遗憾”里至少有一项站得住脚。

### 3. 如果条件不满足怎么办

标准 VIKOR 的常见处理是：

- 如果两个条件都满足，给出唯一妥协解；
- 如果优势条件不满足，给出一个妥协解集合；
- 如果稳定性条件不满足，常把前两名都视为妥协解。

这也是 VIKOR 和很多“只报一个第一名”的方法不一样的地方。

---

## 标准流程

VIKOR 一般按下面 6 步进行：

### 第一步：统一指标方向

把所有指标处理成“越大越好”。

### 第二步：求每个指标上的最好值和最坏值

$$
f_j^* = \max_i f_{ij}, \qquad f_j^- = \min_i f_{ij}.
$$

### 第三步：计算归一化差距

$$
d_{ij} = \frac{f_j^* - f_{ij}}{f_j^* - f_j^-}.
$$

### 第四步：计算 $S_i$ 和 $R_i$

$$
S_i = \sum_{j=1}^n w_j d_{ij}, \qquad
R_i = \max_j (w_j d_{ij}).
$$

### 第五步：计算 $Q_i$

$$
Q_i = v \frac{S_i - S^*}{S^- - S^*} + (1-v)\frac{R_i - R^*}{R^- - R^*}.
$$

### 第六步：按 $Q$ 排序并检验妥协解条件

这一步决定我们最终给出：

- 唯一最优方案；
- 还是一个可接受的妥协解集合。

## 案例讲解：海上风电场址妥协选择

### 问题背景

某沿海地区准备在 4 个备选海域中选择一个海上风电场址。评价指标包括：

- 平均风速：越大越好；
- 并网便利度：越大越好；
- 建设成本：越小越好；
- 生态扰动风险：越小越好；
- 运维可达性：越大越好。

这个问题很适合 VIKOR，因为它天然存在冲突：

- 有的场址风资源很好，但生态风险偏高；
- 有的场址很稳妥，但综合收益没那么突出。

所以我们希望找的是一个**可接受的折中方案**，而不是机械地找某个单一指标第一名。

---

## 可运行 Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "场址": ["W1", "W2", "W3", "W4"],
        "平均风速": [8.4, 8.8, 8.1, 8.6],
        "并网便利度": [82, 90, 78, 86],
        "建设成本": [52, 58, 49, 55],
        "生态扰动风险": [36, 42, 30, 34],
        "运维可达性": [80, 84, 76, 88],
    }
)

weights = pd.Series(
    {
        "平均风速": 0.28,
        "并网便利度": 0.22,
        "建设成本": 0.20,
        "生态扰动风险": 0.15,
        "运维可达性": 0.15,
    }
)
weights = weights / weights.sum()

indicator_types = {
    "平均风速": "benefit",
    "并网便利度": "benefit",
    "建设成本": "cost",
    "生态扰动风险": "cost",
    "运维可达性": "benefit",
}


def positive_transform(df, indicator_types):
    transformed = df.copy()
    for col, col_type in indicator_types.items():
        if col_type == "cost":
            transformed[col] = transformed[col].max() - transformed[col] + transformed[col].min()
    return transformed


def vikor(df, weights, v=0.5):
    best = df.max(axis=0)
    worst = df.min(axis=0)

    denominator = (best - worst).replace(0, np.nan)
    gap = (best - df).div(denominator, axis=1).fillna(0.0)

    weighted_gap = gap.mul(weights, axis=1)
    S = weighted_gap.sum(axis=1)
    R = weighted_gap.max(axis=1)

    S_star, S_minus = S.min(), S.max()
    R_star, R_minus = R.min(), R.max()

    if np.isclose(S_minus, S_star):
        s_term = pd.Series(0.0, index=df.index)
    else:
        s_term = (S - S_star) / (S_minus - S_star)

    if np.isclose(R_minus, R_star):
        r_term = pd.Series(0.0, index=df.index)
    else:
        r_term = (R - R_star) / (R_minus - R_star)

    Q = v * s_term + (1 - v) * r_term
    return gap, S, R, Q


X = positive_transform(data[list(indicator_types.keys())], indicator_types)
gap, S, R, Q = vikor(X, weights, v=0.5)

result = pd.DataFrame(
    {
        "场址": data["场址"],
        "S": S,
        "R": R,
        "Q": Q,
    }
).sort_values("Q").reset_index(drop=True)

m = len(result)
DQ = 1 / (m - 1)
best_name = result.loc[0, "场址"]
second_name = result.loc[1, "场址"]

best_by_S = data.loc[S.idxmin(), "场址"]
best_by_R = data.loc[R.idxmin(), "场址"]

advantage = (result.loc[1, "Q"] - result.loc[0, "Q"]) >= DQ
stability = best_name in {best_by_S, best_by_R}

if advantage and stability:
    compromise_set = [best_name]
elif not advantage:
    compromise_set = result.loc[result["Q"] - result.loc[0, "Q"] < DQ + 1e-12, "场址"].tolist()
else:
    compromise_set = [best_name, second_name]

print("正向化后的数据：")
print(X.round(4))
print("\n归一化差距矩阵：")
print(gap.round(4))
print("\nVIKOR 排序结果：")
print(result.round(4))
print(f"\n可接受优势条件: {advantage}")
print(f"可接受稳定性条件: {stability}")
print(f"妥协解集合: {compromise_set}")
```

---

## 结果应该怎么解释

### 1. $S$

$S$ 小，说明整体表现更接近理想方案。

### 2. $R$

$R$ 小，说明最糟糕的那一项也不至于太差。

### 3. $Q$

$Q$ 是综合之后的妥协指标，越小越好。

### 4. 妥协解集合

这是 VIKOR 很有特色的一点。

如果第一名没有明显拉开差距，或者它虽然 $Q$ 最好，但在 $S$、$R$ 里站不住脚，那么 VIKOR 不一定强行宣布唯一冠军，而可能给出一个妥协解集合。

这比“只报第一名”更符合很多真实决策场景。

---

## VIKOR 和 TOPSIS 的区别

它们都属于基于理想点的多指标决策方法，但关注点不同。

### TOPSIS 更关注“距离”

它看的是方案离正理想解近不近、离负理想解远不远。

### VIKOR 更关注“妥协”

它不仅看整体接近程度，还特别把“最大遗憾”单独拿出来讨论。

所以：

- 如果你更关心几何距离，TOPSIS 更直观；
- 如果你更关心“整体不错，但不要有明显短板”，VIKOR 更合适。

---

## 优缺点与适用场景

### 优点

- 同时考虑整体表现和最差短板；
- 很适合处理存在利益冲突的折中决策；
- 结果解释性强；
- 能给出妥协解，而不只是机械排序。

### 局限

- 需要设定参数 $v$；
- 对权重和极值比较敏感；
- 初学者容易和 TOPSIS 混淆；
- 妥协解判定条件需要解释清楚，否则读者容易只盯着 $Q$ 排名。

### 适用场景

- 多利益相关者决策；
- 需要平衡整体表现与最差风险；
- 不希望某个严重短板被平均分掩盖；
- 选址、供应商选择、项目筛选等妥协型问题。

---

## 一句话总结

VIKOR 的核心不是单纯选“平均最强”的方案，而是选“整体不错、最差短板也能接受”的妥协方案。
