# 因子分析法（FA）

因子分析法（Factor Analysis, FA）和 PCA 很像，但又不能混为一谈。它的核心想法是：许多观测指标背后，往往存在少数几个看不见的“潜在因子”，这些因子才是真正驱动数据相关结构的来源。

如果说 PCA 更像“压缩信息”，那么因子分析更像“寻找隐变量”。

## 这个方法在解决什么问题

因子分析假设，标准化后的观测变量 $Z$ 可以表示成少数公共因子和特殊因子的线性组合：

$$
Z = \Lambda F + \varepsilon
$$

其中：

- $Z$ 是观测变量向量。
- $F$ 是公共因子向量。
- $\Lambda$ 是因子载荷矩阵。
- $\varepsilon$ 是特殊因子或误差项。

若写成协方差结构，可以表示为

$$
\Sigma = \Lambda \Lambda^\top + \Psi
$$

其中 $\Psi$ 是特殊方差构成的对角矩阵。

这意味着：变量之间的相关性，并不只是“巧合”，而是可能由少数共同因子在背后驱动。

## 数学原理

### 1. 公共因子和特殊因子

可以把一个观测指标拆成两部分：

- 公共因子部分：由多个变量共享，反映共同结构。
- 特殊因子部分：只属于这个变量自己，反映个体噪声或独特性。

比如在景区满意度调查中：

- `购票便利`、`指引清晰`、`服务响应` 可能都受到“服务运营水平”这个公共因子影响。
- `景观吸引力`、`活动丰富度`、`环境整洁` 可能都受到“游览体验”这个公共因子影响。

### 2. 因子载荷的含义

因子载荷 $\lambda_{ij}$ 可以理解为：

第 $i$ 个观测指标与第 $j$ 个公共因子的关联强度。

载荷绝对值越大，说明该指标越能代表该因子。

### 3. 共同度

第 $i$ 个指标的共同度定义为

$$
h_i^2 = \sum_{j=1}^q \lambda_{ij}^2
$$

它反映该指标中，有多大比例的方差可以被公共因子解释。

如果共同度很低，说明这个指标和公共因子的联系弱，可能不太适合保留在模型里。

### 4. 为什么要做旋转

刚提取出的因子，数学上可能是对的，但未必好解释。为了让每个指标尽量只在少数因子上有高载荷，常常需要做正交旋转，比如 varimax 旋转。

旋转不会改变模型解释总量，但会让因子的业务含义更清晰。

### 5. 做因子分析之前为什么常看 KMO 和 Bartlett 检验

因为因子分析并不是对任何数据都适用。常见检查有：

- KMO：看变量之间是否适合提取公共因子。
- Bartlett 球形检验：看相关矩阵是否显著偏离单位阵。

一般来说：

- KMO 越接近 1 越好。
- 如果 Bartlett 检验显著，说明变量间相关性足够，不是“彼此毫无关系”的散点。

### 6. 因子分析和 PCA 的区别

两者都能降维，但关注点不同：

- PCA 关心“如何保留总体方差最多的信息”。
- FA 关心“观测变量背后是否有更少的潜在结构”。

所以在解释层面，因子分析通常比 PCA 更适合做“潜在维度”命名。

## 一个直觉 example

假设你做了一份课程满意度问卷，有 8 个题目：

- 讲解清晰。
- 反馈及时。
- 作业合理。
- 答疑充分。
- 课堂氛围。
- 内容有趣。
- 难度适中。
- 收获感。

这些题目看起来很多，但背后可能只是在讲两件事：

- 教学组织。
- 学习体验。

因子分析的工作，就是把这些“藏在后面的大主题”找出来。

## 标准流程

### 第一步：检查是否适合做因子分析

常见做法是看 KMO、Bartlett 检验和相关矩阵。

### 第二步：标准化数据

确保不同量纲变量可比。

### 第三步：提取公共因子

决定提取几个因子，可以参考：

- 特征值大于 1。
- 碎石图。
- 业务背景。

### 第四步：做因子旋转并解释

查看因子载荷矩阵，给每个因子命名。

### 第五步：计算因子得分和综合得分

若题目需要排序，可进一步构造综合得分。

## 建模案例：景区服务质量的潜在结构分析

### 问题背景

现在有 12 个景区，收集了 8 个服务质量指标：

- 购票便利。
- 指引清晰。
- 服务响应。
- 投诉处理。
- 环境整洁。
- 景观吸引。
- 活动丰富。
- 配套完善。

我们怀疑这些指标背后，可能主要由两个潜在因子驱动：

- 因子 1：服务运营能力。
- 因子 2：游览体验质量。

因此尝试用因子分析提取潜在结构，并进一步构造综合得分。

### 建模思路

这个案例里，前四个指标更像“服务流程”，后四个指标更像“游览体验”。如果因子分析合理，旋转后的载荷矩阵就应该把这种结构显示出来。

为了让代码更接近真实使用场景，下面示例包含三件事：

- 计算 KMO，粗略判断数据是否适合做因子分析。
- 用 `FactorAnalysis` 提取 2 个公共因子。
- 用 varimax 旋转增强解释性，再基于因子得分做综合评价。

### Python 代码

```python
import numpy as np
import pandas as pd
from sklearn.decomposition import FactorAnalysis


data = pd.DataFrame(
    {
        "景区": ["A", "B", "C", "D", "E", "F", "G", "H", "I", "J", "K", "L"],
        "购票便利": [9.2, 8.9, 7.6, 7.4, 9.0, 8.7, 6.9, 7.2, 8.5, 7.8, 9.1, 7.0],
        "指引清晰": [8.8, 8.5, 7.2, 7.1, 8.7, 8.4, 6.8, 7.0, 8.2, 7.6, 8.9, 6.9],
        "服务响应": [9.0, 8.7, 7.5, 7.3, 8.8, 8.5, 7.0, 7.1, 8.3, 7.7, 8.9, 7.1],
        "投诉处理": [8.7, 8.4, 7.0, 7.2, 8.6, 8.3, 6.7, 6.9, 8.0, 7.4, 8.8, 6.8],
        "环境整洁": [8.1, 7.9, 9.1, 8.8, 7.0, 7.4, 9.4, 8.9, 8.0, 8.5, 7.6, 9.2],
        "景观吸引": [8.4, 8.2, 9.3, 9.0, 7.2, 7.6, 9.5, 9.1, 8.1, 8.7, 7.8, 9.4],
        "活动丰富": [7.8, 7.5, 8.8, 8.6, 6.8, 7.0, 9.0, 8.7, 7.9, 8.2, 7.3, 8.9],
        "配套完善": [8.0, 7.8, 8.7, 8.5, 7.1, 7.3, 8.9, 8.6, 7.8, 8.3, 7.5, 8.8],
    }
)

feature_cols = [
    "购票便利",
    "指引清晰",
    "服务响应",
    "投诉处理",
    "环境整洁",
    "景观吸引",
    "活动丰富",
    "配套完善",
]

X = data[feature_cols].copy()
Z = (X - X.mean()) / X.std(ddof=1)


def kmo_test(df):
    corr = df.corr().to_numpy()
    inv_corr = np.linalg.inv(corr)

    partial = -inv_corr / np.sqrt(np.outer(np.diag(inv_corr), np.diag(inv_corr)))
    np.fill_diagonal(partial, 0)

    corr_sq = corr**2
    partial_sq = partial**2
    np.fill_diagonal(corr_sq, 0)

    kmo_total = corr_sq.sum() / (corr_sq.sum() + partial_sq.sum())
    kmo_item = corr_sq.sum(axis=0) / (corr_sq.sum(axis=0) + partial_sq.sum(axis=0))
    return kmo_total, pd.Series(kmo_item, index=df.columns)


def varimax(phi, gamma=1.0, q=20, tol=1e-6):
    p, k = phi.shape
    rotation = np.eye(k)
    d_old = 0

    for _ in range(q):
        lam = phi @ rotation
        u, s, vh = np.linalg.svd(
            phi.T @ (lam**3 - (gamma / p) * lam @ np.diag(np.diag(lam.T @ lam)))
        )
        rotation = u @ vh
        d = s.sum()
        if d_old != 0 and abs(d - d_old) < tol:
            break
        d_old = d

    return phi @ rotation, rotation


kmo_total, kmo_item = kmo_test(Z)

fa = FactorAnalysis(n_components=2, random_state=0)
factor_scores = fa.fit_transform(Z)
loadings = fa.components_.T

rotated_loadings, rotation_matrix = varimax(loadings)
rotated_scores = factor_scores @ rotation_matrix

# 统一方向，便于解释
for j in range(rotated_loadings.shape[1]):
    if rotated_loadings[:, j].sum() < 0:
        rotated_loadings[:, j] *= -1
        rotated_scores[:, j] *= -1

explained = (rotated_loadings**2).sum(axis=0)
factor_weights = explained / explained.sum()
composite_score = rotated_scores @ factor_weights

loading_df = pd.DataFrame(
    rotated_loadings,
    index=feature_cols,
    columns=["因子1", "因子2"],
)

score_df = pd.DataFrame(
    rotated_scores,
    index=data["景区"],
    columns=["因子1得分", "因子2得分"],
)

result = pd.DataFrame(
    {
        "景区": data["景区"],
        "综合得分": composite_score,
    }
).sort_values("综合得分", ascending=False)

print("整体 KMO：", round(kmo_total, 4))
print("\n各变量 KMO：")
print(kmo_item.round(4))

print("\n旋转后因子载荷矩阵：")
print(loading_df.round(4))

print("\n因子权重：")
print(pd.Series(factor_weights, index=["因子1", "因子2"]).round(4))

print("\n各景区因子得分：")
print(score_df.round(4))

print("\n综合排序结果：")
print(result.reset_index(drop=True).round(4))
```

### 结果解读

运行后要重点看三部分：

- KMO 是否足够高，说明数据是否适合做因子分析。
- 旋转后因子载荷矩阵，帮助我们给因子命名。
- 因子得分和综合得分，用于排序。

理想情况下，你会看到：

- `购票便利`、`指引清晰`、`服务响应`、`投诉处理` 在同一个因子上载荷较高，这个因子可以解释为“服务运营能力”。
- `环境整洁`、`景观吸引`、`活动丰富`、`配套完善` 在另一个因子上载荷较高，这个因子可以解释为“游览体验质量”。

这就是因子分析最有价值的地方：它不仅给出排序，还把指标背后的潜在结构挖出来了。

## 因子分析法的优缺点与适用场景

### 优点

- 擅长发现观测指标背后的潜在维度。
- 比 PCA 更适合做结构解释和因子命名。
- 可以减少指标冗余，帮助问卷或评价体系简化。
- 在社会科学、管理学、满意度评价中非常常见。

### 局限

- 对样本量和变量相关结构有一定要求。
- 因子个数选择、旋转方式会影响解释结果。
- 结果解释带有一定主观性，需要结合业务背景命名。
- 如果只是单纯想排序，PCA 往往实现更直接。

### 适用场景

- 题目强调“潜在结构”“隐变量”“维度提取”。
- 指标较多，且怀疑背后存在少数公共主题。
- 希望同时做结构解释和综合评价。

## 常见误区

### 1. 把因子分析和 PCA 当成同一种方法

两者都能降维，但目标不同。PCA 重在保留方差，FA 重在解释共同结构。

### 2. 不看载荷矩阵就直接给结论

因子分析最重要的是解释因子含义。如果不分析载荷矩阵，就失去了这类方法的核心价值。

### 3. 因子得分怎么加权完全不说明

如果题目需要综合得分，必须解释权重是怎么来的。示例代码里用的是旋转载荷平方和构造因子权重，这只是教学上常见的一种写法，正式论文中应说明理由。

## 一句话总结

因子分析法的本质是“从许多表面指标中找出少数潜在因子”。它不仅能帮你做排序，更重要的是帮你解释：这些指标背后到底在共同反映什么。
