# CRITIC赋权法

CRITIC 是一种非常有代表性的客观赋权方法，全称常写作 Criteria Importance Through Intercriteria Correlation。它和熵权法有些像，但比熵权法多考虑了一件事：

一个指标不仅要“自己有区分能力”，还要“别和其他指标太重复”。所以，CRITIC 看重两个维度：


- 对比强度：这个指标本身差异大不大。
- 冲突性：这个指标和其他指标是不是高度重复。

## 这个方法在解决什么问题

假设标准化后得到矩阵

$$
Z = (z_{ij})_{m \times n}
$$

对第 $j$ 个指标，定义：

- 标准差 $s_j$，反映该指标的离散程度。
- 相关系数 $r_{jk}$，反映指标 $j$ 与指标 $k$ 的相关性。

CRITIC 定义第 $j$ 个指标的信息量为

$$
C_j = s_j \sum_{k=1}^n (1 - r_{jk})
$$

然后将其归一化得到权重

$$
w_j = \frac{C_j}{\sum_{j=1}^n C_j}
$$

于是：

- $s_j$ 大，说明这个指标区分能力强。
- $\sum (1-r_{jk})$ 大，说明它和其他指标不那么重复，能提供额外信息。
- 两者同时大，权重就高。

## 数学原理

### 1. 标准差反映“能不能拉开差距”

如果一个指标在所有方案之间几乎不变，那么不管它概念上多重要，它在当前样本中都很难帮助排序。

因此，CRITIC 用标准差 $s_j$ 来衡量指标的对比强度。

### 2. 相关性反映“是不是在重复说同一件事”

假设有两个指标：

- 一个是“准时交付率”。
- 一个是“延迟订单占比”。

它们很可能高度相关。如果两个指标几乎在重复表达同一层意思，那么给它们都很高权重就不太合理。

所以，CRITIC 用

$$
1 - r_{jk}
$$

来衡量指标 $j$ 和其他指标的冲突程度。相关性越高，说明重复越强；相关性越低甚至为负，说明提供了不同的信息。

### 3. 为什么要把“离散性”和“冲突性”乘起来

如果一个指标离散很大，但和别的指标完全重复，那它的信息增量有限。

如果一个指标和别人很不一样，但它自己几乎没有变化，也难以区分方案。

因此 CRITIC 认为：真正重要的指标，应当同时满足“自己差异大”和“与别人不太重复”。

### 4. 和熵权法的区别

熵权法主要看“该列内部差异是否明显”。

CRITIC 除了看差异，还会看“该列和其他列是不是高度相关”。

所以在指标之间存在较强相关性时，CRITIC 往往比熵权法更有辨识度。

## 一个直觉 example

假设你在评价供应商，手里有三个指标：

- 交付准时率。
- 订单响应速度。
- 准时交付率。

如果“交付准时率”和“准时交付率”其实几乎是一回事，那么这两个指标就有严重信息重复。

CRITIC 会倾向于降低这种重复指标的权重，把更多权重留给那些既有差异、又能提供新信息的指标。

所以，它很像在回答：

“哪些指标不但自己有用，而且不是在重复别人？”

## 标准流程

### 第一步：正向化

把所有指标统一成“越大越好”。

### 第二步：标准化

把不同量纲拉到同一尺度。

### 第三步：计算标准差

求每一列的离散程度 $s_j$。

### 第四步：计算相关系数矩阵

得到各指标之间的相关性 $r_{jk}$。

### 第五步：计算信息量并归一化

按

$$
C_j = s_j \sum_k (1-r_{jk})
$$

求得信息量，再归一化得到权重。

### 第六步：做后续综合评价

与熵权法类似，CRITIC 本身主要解决赋权问题，后面还需要配合加权求和法或 TOPSIS 做综合排序。

## 建模案例：制造业供应商优选

### 问题背景

某制造企业需要从 5 家候选供应商中选出最优合作方，考虑如下指标：

- 价格水平：越小越好。
- 准时交付率：越大越好。
- 不良率：越小越好。
- 响应速度评分：越大越好。
- 协同研发能力：越大越好。

由于题目没有给权重，而且指标之间可能存在一定相关性，因此采用 CRITIC 赋权，再用加权求和法完成排序。

### 建模思路

这个例子特别适合 CRITIC，因为：

- `准时交付率` 和 `不良率` 可能都在反映管理水平。
- `响应速度` 和 `协同研发能力` 也可能存在联系。
- 我们希望减少信息重复指标被重复加权的问题。

### Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "供应商": ["A", "B", "C", "D", "E"],
        "价格水平": [92, 88, 95, 90, 86],
        "准时交付率": [96, 91, 98, 93, 89],
        "不良率": [1.5, 2.1, 1.2, 1.8, 2.4],
        "响应速度评分": [85, 80, 88, 92, 76],
        "协同研发能力": [78, 82, 75, 90, 84],
    }
)

indicator_types = {
    "价格水平": "cost",
    "准时交付率": "benefit",
    "不良率": "cost",
    "响应速度评分": "benefit",
    "协同研发能力": "benefit",
}


def minmax_to_01(series, indicator_type):
    series = series.astype(float)
    max_val = series.max()
    min_val = series.min()

    if np.isclose(max_val, min_val):
        return pd.Series(np.ones(len(series)), index=series.index)

    if indicator_type == "benefit":
        return (series - min_val) / (max_val - min_val)
    if indicator_type == "cost":
        return (max_val - series) / (max_val - min_val)

    raise ValueError("未知指标类型")


normalized = pd.DataFrame(index=data.index)
for col, indicator_type in indicator_types.items():
    normalized[col] = minmax_to_01(data[col], indicator_type)

std = normalized.std(axis=0, ddof=0)
corr = normalized.corr()
information = std * (1 - corr).sum(axis=1)
weights = information / information.sum()

score = normalized.mul(weights, axis=1).sum(axis=1)

result = data[["供应商"]].copy()
result["综合得分"] = score
result = result.sort_values("综合得分", ascending=False).reset_index(drop=True)

print("标准化矩阵：")
print(normalized.round(4))

print("\n相关系数矩阵：")
print(corr.round(4))

print("\nCRITIC 信息量：")
print(information.round(4))

print("\nCRITIC 权重：")
print(weights.round(4))

print("\n综合排序结果：")
print(result.round(4))
```

### 结果解读

运行后要重点看两张表：

- 相关系数矩阵。
- CRITIC 权重。

如果某个指标：

- 标准差大，说明它本身能拉开供应商差距；
- 与其他指标相关性又不高，说明它提供了独立信息；

那么它在 CRITIC 中的权重就会更高。

这就是 CRITIC 相比熵权法更进一步的地方：它不只是看“差异”，还看“重复不重复”。

## CRITIC 的优缺点与适用场景

### 优点

- 客观赋权，不依赖专家主观打分。
- 同时考虑离散性和相关性，信息利用更丰富。
- 适合处理指标间存在相关关系的场景。
- 很适合和 TOPSIS、加权求和法联用。

### 局限

- 对相关系数比较敏感，样本量太小时可能不稳定。
- 若数据噪声较大，相关性判断可能失真。
- 权重仍然只是“统计意义上的重要性”，不等于政策偏好。
- 相关性只刻画线性关系，复杂非线性重复信息未必能完全识别。

### 适用场景

- 指标之间可能存在明显相关性。
- 希望减少重复指标对结果的重复贡献。
- 题目未给权重，希望做客观赋权。

## 常见误区

### 1. 以为 CRITIC 和熵权法一样

它们都属于客观赋权，但 CRITIC 比熵权法多考虑了指标相关性。

### 2. 忽略负相关的意义

如果两个指标负相关，往往意味着它们在提供不同方向的信息，CRITIC 反而会认为这类指标更有“冲突性”，从而提高信息量。

### 3. 只算权重，不结合排序方法

CRITIC 主要解决的是“怎么赋权”，综合评价还需要后续模型配合。

## 一句话总结

CRITIC 的本质是“既看指标能不能拉开差距，也看它是不是在重复别人”。它比熵权法更强调指标之间的信息独立性。
