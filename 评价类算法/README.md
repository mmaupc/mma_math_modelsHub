# 评价类算法

## 简介

顾名思义，评价某个指标，例如长江的水质、某项任务的完成度、视频的流量、百大up主的含金量等。核心任务在于，将多个指标、多个方案、多个偏好整理成一个可以比较、排序、选优的量化结果。

和最优化问题相比，评价问题更关注：

- 谁更好；
- 为什么更好；
- 差多少；
- 结论是否可解释、可复现。

这一目录整理自 [评价类模型课程3.0.pptx](./评价类模型课程3.0.pptx)。

## 阅读顺序

如果你是第一次系统学习评价类模型，推荐把这一目录理解成三层：

### 第一层：先会一个最基本的评价闭环

先掌握：

1. 指标怎么统一方向；
2. 权重怎么来；
3. 综合得分怎么算；
4. 排序结果怎么解释。

最推荐的起步组合是：

1. [1.加权求和法](./1.加权求和法.md)
2. [3.层次分析法（AHP）](./3.层次分析法（AHP）.md)
3. [5.熵权法](./5.熵权法.md)
4. [4.优劣解距离法（TOPSIS）](./4.优劣解距离法（TOPSIS）.md)

### 第二层：再看不同场景下的扩展方法

当你已经知道基本评价闭环后，再去看↓

- 样本少、信息不完全：GRA；
- 需要妥协解：VIKOR；
- 强调两两偏好：PROMETHEE；
- 强调底线和否决：ELECTRE；
- 强调效率：DEA。

### 第三层：最后看更复杂的结构化方法

例如：

- 主客观权重融合：博弈论组合赋权；
- 模糊语言评价：FCE；
- 标准对照与灰色过渡区：SPA；
- 因果关系网络：DEMATEL；
- 降维与结构提取：PCA、FA。

## 选型速查

如果你面对的是下面这些问题，可以优先考虑：

- 已经知道权重，或者权重容易确定：先看 [加权求和法](./1.加权求和法.md)
- 指标重要性主要依赖专家判断：先看 [AHP](./3.层次分析法（AHP）.md)
- 希望完全基于数据客观赋权：先看 [熵权法](./5.熵权法.md)、[CRITIC](./6.CRITIC赋权法.md)、[变异系数法](./9.变异系数法.md)
- 想做“离理想解有多近”的排序：先看 [TOPSIS](./4.优劣解距离法（TOPSIS）.md)
- 样本少、信息不完整：优先看 [GRA](./10.灰色关联度分析（GRA）.md)
- 方案之间需要妥协解：优先看 [VIKOR](./11.VIKOR法.md)
- 想显式比较方案两两偏好：优先看 [PROMETHEE](./12.PROMETHEE法.md)
- 存在底线、否决项或强约束：优先看 [ELECTRE](./13.ELECTRE法.md)
- 问题本质是投入-产出效率：优先看 [DEA](./14.数据包络分析（DEA）.md)
- 同时想兼顾主观与客观权重：优先看 [博弈论组合赋权](./15.博弈论组合赋权.md)
- 评价结论有明显模糊性：优先看 [FCE](./16.模糊综合评价法（FCE）.md)
- 问题有明确标准、警戒线和灰色过渡区：优先看 [SPA](./17.集对分析（SPA）.md)
- 指标之间存在明显因果影响：优先看 [DEMATEL](./18.DEMATEL法.md)

## 阅读建议

为了让这一目录更容易学，建议每读一篇都尝试回答下面 4 个问题：

1. 这个方法到底在解决什么问题？
2. 它最核心的数学对象是什么？
3. 它和我已经会的方法相比，多解决了什么痛点？
4. 它更适合什么场景，不适合什么场景？

如果你能持续用这 4 个问题去读文档，评价类算法会逐渐变成一套真正能拿来建模的工具箱。
