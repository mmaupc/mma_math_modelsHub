# 加权求和法

加权求和法（Weighted Sum Method, WSM）是多指标综合评价里最基础、也最常用的一类方法。它的想法非常朴素：先把不同指标处理到同一把“尺子”上，再按照权重加总，最后用综合得分排序。

它看起来简单，但并不意味着它“低级”。恰恰相反，很多复杂的评价模型最后也会落到某种加权聚合上。对初学者来说，把加权求和法真正吃透，后面学 TOPSIS、熵权法、CRITIC、组合赋权时都会轻松很多。

## 这个方法在解决什么问题

假设有 $m$ 个待评价对象，$n$ 个评价指标。经过正向化和标准化后，第 $i$ 个对象在第 $j$ 个指标上的值记为 $z_{ij}$，第 $j$ 个指标的权重记为 $w_j$，其中

$$
w_j \ge 0,\quad \sum_{j=1}^n w_j = 1
$$

那么第 $i$ 个对象的综合得分定义为

$$
S_i = \sum_{j=1}^n w_j z_{ij}
$$

通常认为 $S_i$ 越大，说明方案整体越优。

这个模型特别适合下面这类问题：

- 已经有较清晰的指标体系。
- 权重已经给出，或者比较容易确定。
- 希望得到一个解释简单、计算直接、便于写进论文的方法。
- 可以接受“某个指标稍差，可以被其他指标补回来”的补偿逻辑。

## 数学原理

### 1. 为什么不能直接把原始指标相加

原始指标往往不能直接加总，主要有两个原因。

第一，不同指标的方向不一样。比如“利润”是越大越好，“成本”却是越小越好。如果不先统一方向，模型会把高成本误认为高分。

第二，不同指标的量纲不一样。比如“万元”“百分比”“件数”“小时”混在一起，数值大的那一列天然会主导结果，即便它未必更重要。

所以真正关键的工作，不是最后那一步求和，而是求和之前的数据预处理。

### 2. 指标方向统一：把所有指标都变成“越大越好”

常见指标可以分为三类。

- 效益型指标：越大越好，比如利润、满意度、产量。
- 成本型指标：越小越好，比如价格、能耗、投诉率。
- 目标型或区间型指标：离某个目标值越近越好，比如舒适温度、理想心率。

对成本型指标，最常见的正向化写法之一是

$$
x'_{ij} = \frac{\max_i x_{ij} - x_{ij}}{\max_i x_{ij} - \min_i x_{ij}}
$$

这样处理以后，原来“越小越好”的指标就变成了“越大越好”。

对目标型指标，常见处理方式是围绕目标值 $x_j^\ast$ 计算偏差，例如

$$
x'_{ij} = 1 - \frac{|x_{ij} - x_j^\ast|}{\max_i |x_{ij} - x_j^\ast|}
$$

离目标越近，$x'_{ij}$ 就越大。

### 3. 标准化：把不同量纲拉到同一尺度

方向统一之后，还需要做标准化。最常见的是极差标准化：

$$
z_{ij} = \frac{x'_{ij} - \min_i x'_{ij}}{\max_i x'_{ij} - \min_i x'_{ij}}
$$

这样每个指标都被映射到 $[0,1]$ 区间，便于比较和加权。

有些文献也会使用向量标准化：

$$
z_{ij} = \frac{x'_{ij}}{\sqrt{\sum_{i=1}^m (x'_{ij})^2}}
$$

两种写法都很常见。只要你在全文中保持一致，并说明原因即可。

### 4. 权重：决定“哪些指标更重要”

预处理完成后，综合得分写成

$$
S_i = \sum_{j=1}^n w_j z_{ij}
$$

其中权重 $w_j$ 反映的是“第 $j$ 个指标的重要性”。

权重可以来自：

- 题目直接给定。
- 专家判断，比如 AHP。
- 客观数据，比如熵权法、CRITIC。
- 主客观结合的组合赋权。

### 5. 加权求和法的本质是“完全补偿”

这是一个很重要、但初学者常忽略的点。

加权求和法默认：一个指标上的劣势，可以被其他指标上的优势补回来。比如某个项目创新性稍弱，但可实施性和社会效益很高，最终仍然可能排第一。

这叫“补偿性”。如果你的问题不能接受这种补偿，比如任何一项关键安全指标不达标就必须淘汰，那么单纯的加权求和法就不够了，应该先设置约束或门槛。

## 一个直觉 example

假设你要选一台笔记本电脑，只看三个指标：

- 性能，越大越好。
- 续航，越大越好。
- 价格，越小越好。

如果一台电脑性能最强，但特别贵；另一台续航一般，但价格很低；第三台三项都不是第一，却每项都比较均衡，那么加权求和法往往会把第三台推到更靠前的位置。

这件事很像“算总评成绩”：

- 先把平时分、期中、期末、课程项目都换到同一分制。
- 再按比例加总。
- 最后得到总成绩。

所以，加权求和法并不是在“机械加数字”，而是在“统一规则以后做综合聚合”。

## 标准流程

### 第一步：建立评价矩阵

先把 $m$ 个对象、$n$ 个指标整理成原始矩阵

$$
X = (x_{ij})_{m \times n}
$$

并明确每个指标属于效益型、成本型还是目标型。

### 第二步：做正向化

把所有指标统一成“越大越好”的形式，得到处理后的矩阵 $X'$。

### 第三步：做标准化

把不同量纲、不同尺度的指标变到可比较区间，得到标准化矩阵

$$
Z = (z_{ij})_{m \times n}
$$

### 第四步：确定权重

得到权重向量

$$
w = (w_1, w_2, \dots, w_n)^\top
$$

### 第五步：计算综合得分并排序

对每个对象计算

$$
S_i = \sum_{j=1}^n w_j z_{ij}
$$

按 $S_i$ 从大到小排序即可。

## 建模案例：科创项目遴选

### 问题背景

某学院计划从 5 个学生科创项目中遴选 2 个进入校级答辩。评审组给出 5 个指标：

- 创新性：越大越好。
- 落地可行性：越大越好。
- 团队经验：越大越好。
- 社会效益：越大越好。
- 预算压力：越小越好。

经过讨论，评审组认为权重为

$$
w = (0.30,\ 0.25,\ 0.15,\ 0.20,\ 0.10)
$$

现在希望用加权求和法完成排序。

### 建模思路

这个例子非常典型：

- 指标方向并不完全一致，因为 `预算压力` 是成本型指标。
- 指标量纲虽然都写成了分值，但它们代表的含义不同，仍然建议做标准化。
- 权重已经给出，所以我们只需要把“预处理 + 求和”做完整。

### Python 代码

```python
import numpy as np
import pandas as pd


data = pd.DataFrame(
    {
        "项目": ["A-智慧农田", "B-助老陪伴", "C-校园物流", "D-医学影像", "E-低碳社区"],
        "创新性": [92, 80, 85, 95, 88],
        "落地可行性": [75, 90, 88, 70, 82],
        "团队经验": [70, 85, 78, 72, 80],
        "社会效益": [88, 92, 76, 84, 90],
        "预算压力": [60, 45, 50, 75, 55],
    }
)

weights = pd.Series(
    {
        "创新性": 0.30,
        "落地可行性": 0.25,
        "团队经验": 0.15,
        "社会效益": 0.20,
        "预算压力": 0.10,
    }
)

indicator_types = {
    "创新性": "benefit",
    "落地可行性": "benefit",
    "团队经验": "benefit",
    "社会效益": "benefit",
    "预算压力": "cost",
}


def positive_transform(series, indicator_type, target=None):
    series = series.astype(float)
    max_val = series.max()
    min_val = series.min()

    if np.isclose(max_val, min_val):
        return pd.Series(np.ones(len(series)), index=series.index)

    if indicator_type == "benefit":
        return series
    if indicator_type == "cost":
        return max_val - series
    if indicator_type == "target":
        if target is None:
            raise ValueError("target 型指标需要提供目标值")
        distance = (series - target).abs()
        max_distance = distance.max()
        if np.isclose(max_distance, 0):
            return pd.Series(np.ones(len(series)), index=series.index)
        return 1 - distance / max_distance

    raise ValueError("未知指标类型")


def minmax_normalize(series):
    series = series.astype(float)
    max_val = series.max()
    min_val = series.min()
    if np.isclose(max_val, min_val):
        return pd.Series(np.ones(len(series)), index=series.index)
    return (series - min_val) / (max_val - min_val)


processed = pd.DataFrame(index=data.index)
for col, indicator_type in indicator_types.items():
    processed[col] = positive_transform(data[col], indicator_type)

normalized = processed.apply(minmax_normalize, axis=0)
score = normalized.mul(weights, axis=1).sum(axis=1)

result = data[["项目"]].copy()
result["综合得分"] = score
result = result.sort_values("综合得分", ascending=False).reset_index(drop=True)

print("标准化后的指标矩阵：")
print(normalized.round(4))

print("\n综合排序结果：")
print(result.round(4))
```

### 结果解读

运行后你会得到两张关键表：

- 一张是标准化后的指标矩阵，用来检查各指标是否已经处在同一尺度上。
- 一张是综合得分排序表，用来确定最终推荐顺序。

在这个案例里，排在前面的项目通常不是“所有单项都第一”，而是：

- 在高权重指标上表现比较强。
- 没有明显的短板。
- 在成本型指标上没有被严重拖后腿。

这正是加权求和法的特点：它奖励“整体均衡且重点指标不弱”的方案。

## 加权求和法的优缺点与适用场景

### 优点

- 思路直观，初学者最容易理解。
- 代码实现简单，适合快速建模。
- 可解释性强，论文里容易说明。
- 能与 AHP、熵权法、CRITIC 等赋权方法自然结合。

### 局限

- 对权重比较敏感，权重设置不合理会直接影响结果。
- 属于完全补偿型模型，可能掩盖某些关键短板。
- 对正向化、标准化方法有一定敏感性。
- 不能反映方案与“理想解”的几何距离关系，这是 TOPSIS 更擅长的部分。

### 适用场景

- 已经有权重，或者权重较容易获得。
- 目标是做一个透明、稳定、便于解释的综合评分。
- 题目本身不强调“硬约束淘汰”，而更强调综合表现。

## 常见误区

### 1. 直接把原始数据加权相加

这是最常见错误。没有统一方向和量纲的情况下，结果基本不可信。

### 2. 把“权重”和“标准化后的数值范围”混为一谈

标准化是为了消除尺度差异，权重是人为或客观确定的重要性，两者不是一回事。

### 3. 忽略补偿性

如果你的问题里某项指标不达标就必须淘汰，那么应该先加约束，再做综合评分，而不是直接加权求和。

## 一句话总结

加权求和法最像“算总评成绩”：先把各项成绩换到同一把尺子上，再按重要程度求和。它真正的难点不在“加法”，而在指标预处理和权重是否合理。
