# 层次分析法（AHP）

层次分析法（Analytic Hierarchy Process, AHP）是评价类模型里最经典的主观赋权方法之一。它特别适合这样的问题：指标很多、问题很复杂、纯靠客观数据不够，而专家判断又确实很重要。AHP 的核心优势不在“算权重”这件事本身，而在于它把一个复杂决策拆成层次结构，再通过两两比较，把“哪个更重要”这件事量化出来。


## 这个方法在解决什么问题

AHP 通常把问题拆成三层或更多层：

- 目标层：最终要解决的总问题。
- 准则层：评价的主要维度。
- 方案层：候选方案。

对同一层中的元素，构造两两比较判断矩阵

$$
A = (a_{ij})_{n \times n}
$$

其中满足

$$
a_{ij} > 0,\quad a_{ii} = 1,\quad a_{ij} = \frac{1}{a_{ji}}
$$

如果 $a_{ij} = 3$，表示元素 $i$ 相比元素 $j$ “略重要”；如果是 $5$，表示“明显重要”；如果是 $7$ 或 $9$，表示更强的偏好。这就是 Saaty 1-9 标度的基本思想。

理想情况下，判断矩阵满足完全一致性：

$$
a_{ij} a_{jk} = a_{ik}
$$

此时可以提取权重向量

$$
w = (w_1, w_2, \dots, w_n)^\top
$$

并满足

$$
Aw = \lambda_{\max} w
$$

其中 $\lambda_{\max}$ 是矩阵最大特征值。

## 数学原理

### 1. 为什么要做两两比较

如果你直接问专家：“交通、成本、安全、覆盖能力的权重分别是多少？”

很多人会觉得很难回答。因为人脑更擅长处理的是“局部比较”，比如：

- 覆盖能力和交通便利，哪个更重要？
- 如果更重要，大概强多少？

所以 AHP 的聪明之处在于：它不要求专家一步给出所有权重，而是先把整体问题拆开，让专家做一系列两两比较，再从这些比较中提取出整体权重。

### 2. 权重提取：最大特征值对应的特征向量

对判断矩阵 $A$，AHP 的经典做法是取最大特征值 $\lambda_{\max}$ 对应的特征向量，再归一化得到权重：

$$
w_j = \frac{v_j}{\sum_{k=1}^n v_k}
$$

这里 $v$ 是 $A$ 对应最大特征值的特征向量。

这一步的直觉是：如果某个元素在与其他元素的比较中长期占优，那么它在主特征向量中的权重通常就会更大。

### 3. 一致性检验：防止判断自相矛盾

AHP 最有价值的一点，是它不会无条件相信专家判断，而是会检查这些判断是否前后矛盾。

一致性指标定义为

$$
CI = \frac{\lambda_{\max} - n}{n - 1}
$$

一致性比例定义为

$$
CR = \frac{CI}{RI}
$$

其中 $RI$ 是随机一致性指标。通常认为：

- 如果 $CR < 0.10$，则一致性可以接受。
- 如果 $CR \ge 0.10$，则判断矩阵需要重新调整。

这相当于在问：专家的比较虽然允许不完美，但至少不能明显自相矛盾。

### 4. 层次总排序

准则层有权重，方案层在每个准则下也有局部权重。最后把这些局部权重逐层合成，就得到方案层的总权重。

如果第 $k$ 个准则权重为 $w_k$，方案 $i$ 在该准则下的局部权重为 $p_{ik}$，则总权重可写为

$$
P_i = \sum_k w_k p_{ik}
$$

本质上，AHP 是“先比较，再赋权，再层层汇总”。

## 一个直觉 example

假设你要选一个宿舍，不考虑太多数学细节，真实思路大概是这样的：

- 先想清楚要看哪些维度，比如距离、价格、环境、安全。
- 然后不断做比较，比如“距离和价格谁更重要？”“环境和安全谁更重要？”
- 最后脑子里其实会形成一套偏好顺序。

AHP 做的就是把这套偏好顺序写成矩阵，再用数学方法把它提取成权重。

所以它并不是“凭空算出权重”，而是“把人的判断组织起来”。

## 标准流程

### 第一步：搭建层次结构

明确目标层、准则层、方案层，必要时还可以加入子准则层。

### 第二步：构造判断矩阵

对同一层元素进行两两比较，形成正互反矩阵。

### 第三步：计算权重

可采用特征值法，也可采用几何平均法。教材和竞赛里最常见的是特征值法。

### 第四步：做一致性检验

计算 $\lambda_{\max}$、$CI$、$CR$，判断矩阵是否可靠。

### 第五步：层次总排序

将局部权重逐层向上汇总，得到方案层的总权重。

## 建模案例：应急物资仓库选址

### 问题背景

某市准备在 3 个候选片区中选择一个应急物资仓库。决策层认为需要重点考虑 4 个准则：

- 覆盖能力。
- 交通便利。
- 建设成本。
- 安全冗余。

由于这类问题很难完全依靠客观数据决策，因此非常适合使用 AHP 做主观赋权。

### 建模思路

这个案例分两层：

- 先确定准则层权重。
- 再确定每个准则下，三个候选片区的局部权重。

为了把注意力放在 AHP 的计算流程上，下面代码使用“由优先级自动构造的正互反矩阵”作为教学示例。这样做的好处是：

- 代码更容易看懂。
- 一致性会天然较好。
- 初学者更容易先吃透算法流程，再回头处理真实专家打分。

### Python 代码

```python
import numpy as np
import pandas as pd


RI = {
    1: 0.00,
    2: 0.00,
    3: 0.58,
    4: 0.90,
    5: 1.12,
    6: 1.24,
    7: 1.32,
    8: 1.41,
    9: 1.45,
    10: 1.49,
}


def ratio_matrix(priority):
    priority = np.array(priority, dtype=float)
    return priority[:, None] / priority[None, :]


def ahp_weight(matrix):
    eigvals, eigvecs = np.linalg.eig(matrix)
    idx = np.argmax(eigvals.real)
    lambda_max = eigvals[idx].real
    weight = np.abs(eigvecs[:, idx].real)
    weight = weight / weight.sum()

    n = matrix.shape[0]
    if n <= 2 or np.isclose(RI[n], 0):
        ci = 0.0
        cr = 0.0
    else:
        ci = (lambda_max - n) / (n - 1)
        cr = ci / RI[n]

    return weight, lambda_max, ci, cr


criteria = ["覆盖能力", "交通便利", "建设成本", "安全冗余"]
alternatives = ["东站片区", "北郊片区", "高新区片区"]

# 准则层优先级：覆盖能力 > 交通便利 > 建设成本 ≈ 安全冗余
criteria_matrix = ratio_matrix([8, 6, 3, 3])

# 方案层优先级
local_priority = {
    "覆盖能力": [5, 3, 2],
    "交通便利": [2, 4, 2],
    "建设成本": [2, 5, 3],  # 这里的数值越大表示越“值得选”，已经隐含了低成本偏好
    "安全冗余": [3, 2, 5],
}

criteria_weight, lambda_max, ci, cr = ahp_weight(criteria_matrix)
print("准则层权重：")
print(pd.Series(criteria_weight, index=criteria).round(4))
print(f"\n准则层一致性检验：lambda_max={lambda_max:.4f}, CI={ci:.4f}, CR={cr:.4f}")

local_weight_dict = {}
for criterion, priority in local_priority.items():
    matrix = ratio_matrix(priority)
    weight, lmbd, ci_value, cr_value = ahp_weight(matrix)
    local_weight_dict[criterion] = weight

    print(f"\n{criterion} 下的局部权重：")
    print(pd.Series(weight, index=alternatives).round(4))
    print(f"一致性检验：lambda_max={lmbd:.4f}, CI={ci_value:.4f}, CR={cr_value:.4f}")

local_weight_df = pd.DataFrame(local_weight_dict, index=alternatives)
final_score = local_weight_df.to_numpy() @ criteria_weight

result = pd.DataFrame(
    {
        "方案": alternatives,
        "综合权重": final_score,
    }
).sort_values("综合权重", ascending=False)

print("\n方案层局部权重矩阵：")
print(local_weight_df.round(4))

print("\n最终排序结果：")
print(result.reset_index(drop=True).round(4))
```

### 结果解读

这个例子会输出三类核心结果：

- 准则层权重，表示决策层最看重哪些维度。
- 各准则下的方案局部权重，表示每个片区在单一维度上的优劣。
- 最终总权重，用于综合排序。

如果你看到：

- `覆盖能力` 权重最高，说明决策层更重视应急辐射能力。
- 某个片区在 `建设成本` 下表现最好，但总排序仍未必第一，说明 AHP 看的是全局综合平衡。
- `CR` 很小，说明判断矩阵前后基本一致。

这正是 AHP 的特点：它把主观判断变成了可检验、可解释、可复现的计算过程。

## AHP 的优缺点与适用场景

### 优点

- 特别适合专家经验重要、客观数据不足的场景。
- 结构清晰，容易写成层次化分析过程。
- 有一致性检验，比简单“拍脑袋定权重”更严谨。
- 能与 TOPSIS、模糊综合评价等方法联用。

### 局限

- 结果依赖专家判断质量，主观性较强。
- 指标一多，判断矩阵数量会快速增加。
- 不同专家意见差异大时，需要做群决策整合。
- 如果强行给出不合理的判断，可能通不过一致性检验。

### 适用场景

- 指标里有大量定性因素。
- 题目强调“偏好”“重要性”“层次结构”。
- 希望得到一套有逻辑链条的主观权重。

## 常见误区

### 1. 只会算权重，不会解释层次结构

AHP 的亮点不仅是结果，更是“为什么这样分层、为什么这样比较”的逻辑。

### 2. 忽略一致性检验

如果不看 $CR$，那 AHP 就退化成了普通的主观打分。

### 3. 把所有问题都硬套 AHP

如果数据很充分、客观性很强，未必一定要用 AHP。此时熵权法、CRITIC 等客观赋权方法可能更自然。

## 一句话总结

AHP 的本质不是“神奇地算出权重”，而是把复杂决策拆开，再通过两两比较把人的主观判断变成可计算、可检验的权重体系。
