# 差分进化算法（DE）

差分进化（Differential Evolution, DE）是一种**面向连续优化问题**的群体智能启发式算法。

它和遗传算法一样，也会维护一组候选解，但它的“进化”方式并不是传统意义上的“父母交叉生孩子”，而是直接利用**个体之间的差值**来构造新的搜索方向。

简单说，DE 的核心不是“挑谁繁殖”，而是：

- 看看不同个体之间到底差了多少；
- 把这种差值当成搜索方向；
- 再用贪心选择保留更优的新解。

它很适合下面这类问题：

- 连续变量优化问题；
- 非线性、非凸、多峰函数优化；
- 难以直接求导的问题；
- 数学建模中的参数估计、连续选址、控制参数寻优等问题。

## 问题定义

我们考虑如下连续优化问题：

$$
\min_{x \in \Omega} f(x)
$$

其中：

- $x = (x_1, x_2, \dots, x_D)^T$ 是待优化变量；
- $D$ 是决策变量维数；
- $f(x)$ 是目标函数；
- $\Omega$ 是可行域，常写成变量边界：

$$
\Omega = \{x \in \mathbb{R}^D \mid l_j \le x_j \le u_j,\ j=1,2,\dots,D\}
$$

DE 会维护一个大小为 $NP$ 的种群：

$$
P^{(g)} = \{x_1^{(g)}, x_2^{(g)}, \dots, x_{NP}^{(g)}\}
$$

其中每个个体 $x_i^{(g)}$ 都是一个 $D$ 维实数向量。

## 核心机制

DE 最核心的三步是：

1. 变异（Mutation）
2. 交叉（Crossover）
3. 选择（Selection）

### 1. 变异（Mutation）

以最经典的 **DE/rand/1** 为例，对第 $i$ 个目标个体 $x_i^{(g)}$，随机选出三个互不相同的个体：

$$
x_{r1}^{(g)},\quad x_{r2}^{(g)},\quad x_{r3}^{(g)}
$$

然后构造变异向量：

$$
v_i^{(g)} = x_{r1}^{(g)} + F \cdot \left(x_{r2}^{(g)} - x_{r3}^{(g)}\right)
$$

其中：

- $F$ 是缩放因子；
- $x_{r2}^{(g)} - x_{r3}^{(g)}$ 体现了种群内部的差异方向。

### 2. 交叉（Crossover）

有了变异向量 $v_i^{(g)}$ 后，再和原个体 $x_i^{(g)}$ 做交叉，生成试验向量 $u_i^{(g)}$：

$$
u_{ij}^{(g)}=
\begin{cases}
v_{ij}^{(g)}, & \text{if } rand_j \le CR \text{ or } j=j_{\mathrm{rand}} \\
x_{ij}^{(g)}, & \text{otherwise}
\end{cases}
$$

其中：

- $CR \in [0,1]$ 是交叉概率；
- $j_{\mathrm{rand}}$ 保证至少有一维来自变异向量。

### 3. 选择（Selection）

DE 的选择非常直接，就是“谁更优谁留下”：

$$
x_i^{(g+1)}=
\begin{cases}
u_i^{(g)}, & \text{if } f(u_i^{(g)}) \le f(x_i^{(g)}) \\
x_i^{(g)}, & \text{otherwise}
\end{cases}
$$

这一步带有明显的贪心味道。

## 人话

如果把每个个体看成一个方案，那么 DE 的思路不是“让两个方案生孩子”，而是：

1. 先找三个已有方案；
2. 看看后两个方案到底差在哪；
3. 把这种差异方向加到第一个方案上；
4. 造出一个新方案试试看。

所以 DE 的“创新”并不是凭空乱想，而是利用当前种群内部已经存在的结构差异来产生新解。

这也是为什么它在连续变量问题上常常很自然，因为“差值”这件事本身就有几何意义。

## 基本流程

### 初始化

对每个个体、每个维度随机初始化：

$$
x_{ij}^{(0)} = l_j + rand_{ij}(u_j-l_j)
$$

### 对每一代重复以下步骤

#### 第一步：变异

利用差分向量生成变异向量 $v_i^{(g)}$。

#### 第二步：交叉

由目标个体和变异向量生成试验向量 $u_i^{(g)}$。

#### 第三步：边界处理

如果新向量超出边界，可采用：

- 截断；
- 随机重置；
- 反射回可行域。

#### 第四步：选择

比较新旧两个解，保留目标函数值更优的那个。

#### 第五步：判断是否终止

例如：

- 达到最大迭代次数；
- 已达到满意精度；
- 连续若干代提升极小。

## 建模例题：连续平面双站点选址优化

### 问题背景

假设现在有 10 个需求点，每个点都有坐标和需求权重。我们希望在平面上建设两个服务站点，使得所有需求点都被分配到更近的站点，总加权距离尽量小。

此外，为了避免两个站点建得过近，我们要求站点间距不能太小，否则加入罚项。

这个问题的特点是：

- 决策变量是连续坐标；
- 目标函数没有现成梯度；
- 局部最优并不少；
- 非常适合用 DE 来直接搜索。

### Python 代码

```python
import random
import math
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimSun']
plt.rcParams['axes.unicode_minus'] = False


# 需求点：(x, y, weight)
points = [
    (10, 15, 8),
    (18, 30, 6),
    (22, 12, 7),
    (30, 28, 5),
    (35, 40, 9),
    (45, 18, 6),
    (52, 35, 8),
    (60, 22, 7),
    (68, 38, 5),
    (75, 15, 9),
]


def objective(vec):
    x1, y1, x2, y2 = vec
    total = 0.0

    dist_12 = math.hypot(x1 - x2, y1 - y2)
    penalty = 0.0
    if dist_12 < 12:
        penalty += 1000 * (12 - dist_12)

    for px, py, w in points:
        d1 = math.hypot(px - x1, py - y1)
        d2 = math.hypot(px - x2, py - y2)
        total += w * min(d1, d2)

    return total + penalty


def clip(vec, lower, upper):
    return [max(lower[i], min(upper[i], vec[i])) for i in range(len(vec))]


def differential_evolution(dim, pop_size, generations, F, CR, lower, upper, seed=42):
    random.seed(seed)

    pop = [
        [random.uniform(lower[j], upper[j]) for j in range(dim)]
        for _ in range(pop_size)
    ]

    best = min(pop, key=objective)
    best_val = objective(best)
    history = []

    for _ in range(generations):
        new_pop = []

        for i in range(pop_size):
            ids = list(range(pop_size))
            ids.remove(i)
            r1, r2, r3 = random.sample(ids, 3)

            x1, x2, x3 = pop[r1], pop[r2], pop[r3]

            v = [x1[j] + F * (x2[j] - x3[j]) for j in range(dim)]
            v = clip(v, lower, upper)

            u = pop[i][:]
            j_rand = random.randrange(dim)
            for j in range(dim):
                if random.random() < CR or j == j_rand:
                    u[j] = v[j]

            if objective(u) <= objective(pop[i]):
                new_pop.append(u)
            else:
                new_pop.append(pop[i])

        pop = new_pop
        cur_best = min(pop, key=objective)
        cur_best_val = objective(cur_best)

        if cur_best_val < best_val:
            best = cur_best[:]
            best_val = cur_best_val

        history.append(best_val)

    return best, best_val, history


def plot_result(best, history):
    x1, y1, x2, y2 = best

    plt.figure(figsize=(12, 5))

    plt.subplot(1, 2, 1)
    plt.plot(history)
    plt.xlabel('迭代代数')
    plt.ylabel('当前最优目标值')
    plt.title('DE 收敛过程')
    plt.grid(alpha=0.3)

    plt.subplot(1, 2, 2)
    for px, py, w in points:
        plt.scatter(px, py, s=25 * w)
        plt.text(px + 0.8, py + 0.8, f'w={w}', fontsize=8)

    plt.scatter([x1, x2], [y1, y2], marker='s', s=140)
    plt.text(x1 + 1, y1 + 1, 'Station 1')
    plt.text(x2 + 1, y2 + 1, 'Station 2')

    for px, py, _ in points:
        d1 = math.hypot(px - x1, py - y1)
        d2 = math.hypot(px - x2, py - y2)
        if d1 <= d2:
            plt.plot([px, x1], [py, y1], alpha=0.4)
        else:
            plt.plot([px, x2], [py, y2], alpha=0.4)

    plt.xlabel('X')
    plt.ylabel('Y')
    plt.title('双站点选址结果')
    plt.grid(alpha=0.3)
    plt.tight_layout()
    plt.show()


if __name__ == '__main__':
    lower = [0, 0, 0, 0]
    upper = [80, 50, 80, 50]

    best, best_val, history = differential_evolution(
        dim=4,
        pop_size=40,
        generations=200,
        F=0.7,
        CR=0.9,
        lower=lower,
        upper=upper,
        seed=42
    )

    print('最优站点坐标：', best)
    print('最优目标值：', best_val)
    plot_result(best, history)
```

### 结果解读

运行后通常会得到两幅图：

1. 左图是收敛曲线，展示最优目标值如何逐步下降；
2. 右图是站点布局图，能直观看出两个站点如何分担需求点。

这个例子非常适合放进数学建模文档，因为它能很好体现 DE 对连续变量黑箱优化问题的适应性。

## 差分进化的优缺点及适用场景

### 优点

- 结构简单，容易自己手写实现；
- 对连续变量优化很自然；
- 差分方向来自种群本身，具有较强自适应味道；
- 在很多黑箱优化问题中效果稳定。

### 局限

- 参数 $F$、$CR$、种群规模都会影响效果；
- 对边界和约束的处理要额外设计；
- 对离散组合问题通常不是最自然的选择；
- 在高维复杂问题中也可能早熟。

### 适用场景

- 连续参数寻优；
- 非线性参数估计；
- 连续选址与布局优化；
- 难以求导的黑箱优化；
- 多峰函数的全局搜索。

## 一句话总结

差分进化最有代表性的地方，不是“随机”本身，而是它把**个体之间的差异**直接拿来当作搜索方向。

这使得它在连续优化问题中往往又简单、又有效。
