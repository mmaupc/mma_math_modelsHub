
## 现代启发式算法入门

### 定义
“启发式算法”是相对于最优化方法提出的，与传统最优化方法一样，其根本目的是找到某个问题的最优解。

不同之处在于：传统方法往往依赖较强的数学条件，例如目标函数可导、约束结构清晰、问题形式规整；而启发式算法更适合面对那些结构复杂、难以解析、难以求导、甚至只能通过仿真或黑箱方式评估的问题。

启发式算法的定义之一是：一个基于直观或经验构造的算法，在可接受的花费（指计算时间和空间）下给出待解决组合优化问题每一个实例的一个可行解，该可行解与最优解的偏离程度一般不能被预计。

在常见的现代启发式算法中，很多方法都受到自然现象的启发，例如蚁群算法、模拟退火、粒子群算法、遗传算法等。

### 为什么要学启发式算法

如果你学过《最优化方法》，老师会教你单纯形法、牛顿法、外点法梯度下降等方法，归根结底这些方法都是为了**找到某个函数的最低点** ，这个过程可以被简单地称为“优化”。在可导优化问题中，我们常常借助梯度信息来寻找更优解。梯度表示函数值增长最快的方向，因此在求最小值时，通常沿着负梯度方向迭代下降。

![](image-1.png)

因为我们要求最小值，所以我们只需要在梯度之前加一个负号，就可以知道最低点在什么方向了。

![](image-2.png)

求最小值是一个很基础但是可以被泛化的路径，因为现实世界中几乎所有问题，都可以被抽象为“让某个指标更好”。例如在商业中，我们需要让成本 $cost(x)$ 最小，并且让利润的负数 $-profit(x)$ 尽可能小。在物流领域中，我们希望让运输路径的总距离 $f(x)$ 最小。如果你研究过AI，那么你肯定知道深度学习的过程是通过不断调整参数，让预测值和真实值的误差（Loss）$L(x)$ 最小。哪怕你有很多个目标，也可以通过某个指标（由其它指标的函数组合构成）的抽象（或是说构造一个综合指标），将它们统一到同一个优化框架下，从而寻求最优解或可接受的近似解。因此，你可以把优化这门学科视为“万物皆数”一般，建立起“万物皆可优化”的思维。

但是求最小值（或是说最优值）并不是一个容易的事情。对于结构良好、可导、可解析的问题，传统最优化方法通常效率更高、理论也更扎实；但现实世界，尤其是数学建模问题，往往比课本上的习题要复杂和脏得多。例如很多时候你的 $f(x)$ 并没有显式解析表达式，而只能通过仿真、实验或黑箱过程得到其函数值；或是传统方法很容易陷入局部最优点，往任何方向走都是向上，但是真正的最低点还在几百单位之外；又或是问题是组合优化问题，例如去10个城市，怎么排序路径最短，这个时候传统的办法就不那么好用了。

总而言之，如果说《最优化方法》是如何在完美的F1地图上快速找到最晚刹车点，那么《启发式算法》就是教你如何在没有地图、地形复杂的荒野中，靠一群探险队员的协作找到一个足够安全且低的四号谷地。它们通常不保证全局最优，但常常能在可接受的计算代价下给出质量不错的结果，这也是它们在工程优化、组合优化和黑箱优化中长期受到重视的原因。

### 本文档会接受哪些算法

本项目不会试图罗列所有“名字很新”的启发式算法，而是选择几种具有代表性、教学价值高、易于上手的方法，帮助读者建立对这一类算法的整体认识。

本文档主要介绍以下算法：

1. 模拟退火（单解轨迹型）
2. 遗传算法（进化式）
3. 粒子群算法（群体智能）
4. 差分进化算法
5. 蚁群优化算法（群体智能）
6. 灰狼优化算法（群体智能）
7. 协方差矩阵自适应进化策略
8. 贝叶斯优化
9.  代理模型/昂贵优化的 surrogate-assisted evolutionary algorithms
10. 机器学习增强的 metaheuristics

启发式算法的发展非常快，新算法层出不穷，尤其近些年出现了大量以自然现象、动物行为或群体协作为灵感的元启发式算法。但是从教学的角度看，真正重要的并不只是“又认识了一种新算法”，而是理解这些算法背后的共性问题：

- 如何表示一个解？

- 如何生成新解？

- 如何在“全局探索”和“局部开发”之间取得平衡？

- 如何避免过早收敛？

- 如何根据问题类型选择合适的算法？

上面这些算法基本覆盖了启发式优化中几种重要思路：单解搜索、进化搜索、群体协同搜索、离散路径搜索，以及近年较流行的新型群智算法。

### 这些算法是否就是“最前沿、最好用”的答案？

不完全是。本文档介绍的算法更适合作为教学与入门的代表，而不意味着它们在所有问题上都是最先进、最好用的选择。在实际研究和工程应用中，一个算法是否有效，往往高度依赖于问题类型、变量维度、约束形式、评价成本以及参数设置。

从实践角度看，除了本文档中的算法外，下面这些方向也非常值得进一步了解：

- CMA-ES：在连续黑箱优化中具有很强的代表性。

- 贝叶斯优化（Bayesian Optimization）：适合目标函数评估代价很高的场景。

- 代理模型优化 / 昂贵优化：适用于仿真、实验驱动问题。

- 机器学习增强的元启发式算法：是近年来较值得关注的发展方向之一。

因此，我们也会在文档中介绍它们，形成一个基础的认知。

### 建议的阅读方式

如果你是第一次接触这类方法，建议按如下顺序阅读：

**模拟退火 → 遗传算法 → 粒子群算法 → 蚁群算法 → 灰狼优化 / 鲸鱼优化 / 麻雀搜索算法**

这样的顺序的好处是，先从思想最直观、结构最清晰的算法入手，再逐步过渡到现代群体智能算法，便于比较它们之间的相似与差异。

如果你已经学过一些最优化方法，那么可以重点关注这些启发式算法在以下场景中的优势：无法求导、目标函数为黑箱、存在大量局部最优、变量是离散或组合结构、问题评价过程复杂且不规则...
