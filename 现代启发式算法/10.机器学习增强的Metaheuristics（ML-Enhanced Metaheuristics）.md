# 机器学习增强的 Metaheuristics（ML-Enhanced Metaheuristics）

“机器学习增强的元启发式算法”并不是某一个固定算法名，而是一类方法方向。

它的核心思想是：在原有启发式算法或群体智能算法的基础上，引入机器学习模块，让算法不只是盲目试错，而是能从历史搜索过程中学到一些东西，比如：

- 哪些区域更像有好解；
- 哪些个体更值得保留；
- 哪种操作符在当前阶段更有效；
- 哪些解更可能可行。

所以它和单纯的“随机搜索 + 保留最好解”相比，多了一层“边搜边学”的味道。

## 问题定义

我们仍然考虑优化问题：

$$
\min_{x \in \Omega} f(x)
$$

不同的是，我们会在搜索过程中额外维护一个学习模型：

$$
\hat{f}_t(x) = g(x; \mathcal{D}_t)
$$

其中：

- $\mathcal{D}_t$ 是截至第 $t$ 轮的历史搜索数据；
- $g(\cdot)$ 可以是回归模型、分类模型、排序模型等。

但这里要注意：**ML-enhanced 不一定等于代理模型替代真实评估。**

很多时候，机器学习模块只是负责：

- 生成更有希望的新样本；
- 预测可行区域；
- 动态调节参数；
- 选择更合适的搜索策略。

## 核心机制

这类方法的共同思路通常是：

1. 用元启发式算法产生搜索数据；
2. 用机器学习模型从历史数据中学习规律；
3. 再反过来指导下一轮搜索。

这里给出一个代表性的“回归引导搜索”写法。

### 1. 学习一个经验模型

比如用一个二次回归模型去拟合历史数据：

$$
\hat{f}_t(x) = w_0 + \sum_i w_i x_i + \sum_i w_{ii}x_i^2 + \sum_{i<j} w_{ij}x_i x_j
$$

这个模型并不是为了完全替代真实目标函数，而是为了给出一个“哪里更像有希望”的经验判断。

### 2. 用模型构造引导中心

假设我们在当前搜索空间中生成一批候选点，并用学习模型做预测。取预测值最好的前 $K$ 个点，可构造一个学习引导中心：

$$
\mu_t = \frac{1}{K} \sum_{x \in \mathrm{TopK}(\hat{f}_t)} x
$$

### 3. 围绕引导中心采样

然后在这个中心附近生成一批候选解：

$$
x_{\mathrm{new}} = \mu_t + \sigma_t \odot \epsilon,\quad \epsilon \sim \mathcal{N}(0, I)
$$

与此同时，普通的进化操作仍然继续保留。

所以本质上是：

- 一部分个体继续靠传统元启发式搜索；
- 一部分个体由学习模型做“定向推荐”。

## 人话

你可以把它理解成：原来的元启发式算法像一支靠经验在野外找矿的队伍，而机器学习模块像一个随队的数据分析员。

前几天大家到处挖，分析员就在旁边记：

- 哪片区域常出好矿；
- 哪种地形更靠谱；
- 哪些方向白跑概率大。

后面队伍再出发时，就不只是“凭感觉乱找”，而是会参考这份越来越靠谱的经验笔记。

所以 ML-enhanced 的重点，不是把元启发式推翻重来，而是给它加上一层“从历史中学习”的能力。

## 基本流程

### 初始化

- 初始化种群；
- 真实评估初始个体；
- 建立历史样本库。

### 对每一代重复以下步骤

#### 第一步：训练学习模型

用历史样本拟合一个轻量回归或分类模型。

#### 第二步：生成候选个体

一部分通过普通进化操作产生，一部分通过学习引导策略产生。

#### 第三步：真实评估

对新个体仍然用真实目标函数做评价。

#### 第四步：更新种群与样本库

保留优秀个体，并把新数据加入历史库。

#### 第五步：判断是否终止

例如达到最大迭代次数或收敛到满意结果。

## 建模例题：学习引导的二维非凸优化

### 问题背景

我们考虑一个带禁区罚项的二维非凸优化问题。目标函数选用 Himmelblau 函数，并把其中一个区域设置为“不可取区域”，从而模拟实际建模中常见的复杂可行域。

原函数为：

$$
f(x,y) = (x^2 + y - 11)^2 + (x + y^2 - 7)^2
$$

然后对禁区加入罚项。

这个例子的目的不是追求最前沿性能，而是演示一种很典型的 ML-enhanced 思路：

- 基础搜索仍然是进化式的；
- 机器学习模型只负责帮你判断“下一步往哪片区域试更像有戏”；
- 所有候选解仍然做真实评估。

### Python 代码

```python
import random
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimSun']
plt.rcParams['axes.unicode_minus'] = False


def objective(vec):
    x, y = vec
    base = (x * x + y - 11) ** 2 + (x + y * y - 7) ** 2

    penalty = 0.0
    dist = np.sqrt((x - 3.0) ** 2 + (y - 2.0) ** 2)
    if dist < 1.4:
        penalty += 200 * (1.4 - dist) ** 2

    return base + penalty


def feature_map(X):
    X = np.atleast_2d(X)
    x = X[:, 0]
    y = X[:, 1]
    return np.column_stack([
        np.ones(len(X)),
        x,
        y,
        x ** 2,
        x * y,
        y ** 2
    ])


def fit_quadratic_model(X, y, reg=1e-6):
    Z = feature_map(X)
    A = Z.T @ Z + reg * np.eye(Z.shape[1])
    coef = np.linalg.solve(A, Z.T @ y)
    return coef


def predict_quadratic(X, coef):
    Z = feature_map(X)
    return Z @ coef


def crossover(p1, p2):
    alpha = random.random()
    return alpha * p1 + (1 - alpha) * p2


def mutate(x, scale):
    return x + np.random.randn(2) * scale


def learning_guided_ea(pop_size=30, generations=80, seed=42):
    random.seed(seed)
    np.random.seed(seed)

    lower = np.array([-6.0, -6.0])
    upper = np.array([6.0, 6.0])

    population = lower + np.random.rand(pop_size, 2) * (upper - lower)
    fitness = np.array([objective(x) for x in population])

    archive_X = population.copy()
    archive_y = fitness.copy()

    best_idx = np.argmin(fitness)
    best_x = population[best_idx].copy()
    best_y = fitness[best_idx]
    history = [best_y]
    guide_path = []

    for g in range(generations):
        order = np.argsort(fitness)
        elites = population[order[:6]]

        coef = fit_quadratic_model(archive_X, archive_y)

        probe = lower + np.random.rand(200, 2) * (upper - lower)
        pred = predict_quadratic(probe, coef)
        top_pred = probe[np.argsort(pred)[:12]]

        guide_center = np.mean(top_pred, axis=0)
        guide_spread = np.std(top_pred, axis=0) + 0.25
        guide_path.append(guide_center.copy())

        offspring = []
        scale = 0.7 * (1 - g / generations) + 0.15

        while len(offspring) < pop_size:
            if random.random() < 0.6:
                p1 = elites[random.randrange(len(elites))]
                p2 = elites[random.randrange(len(elites))]
                child = crossover(p1, p2)
                child = mutate(child, scale)
            else:
                child = guide_center + np.random.randn(2) * guide_spread

            child = np.clip(child, lower, upper)
            offspring.append(child)

        offspring = np.array(offspring)
        off_fit = np.array([objective(x) for x in offspring])

        combined_X = np.vstack([population, offspring])
        combined_y = np.concatenate([fitness, off_fit])
        keep = np.argsort(combined_y)[:pop_size]

        population = combined_X[keep]
        fitness = combined_y[keep]

        archive_X = np.vstack([archive_X, offspring])
        archive_y = np.concatenate([archive_y, off_fit])

        if fitness[0] < best_y:
            best_y = fitness[0]
            best_x = population[0].copy()

        history.append(best_y)

    return best_x, best_y, history, archive_X, np.array(guide_path)


def plot_result(best_x, history, archive_X, guide_path):
    xs = np.linspace(-6, 6, 240)
    ys = np.linspace(-6, 6, 240)
    Xg, Yg = np.meshgrid(xs, ys)
    Z = (Xg ** 2 + Yg - 11) ** 2 + (Xg + Yg ** 2 - 7) ** 2

    dist = np.sqrt((Xg - 3.0) ** 2 + (Yg - 2.0) ** 2)
    Z = Z + np.where(dist < 1.4, 200 * (1.4 - dist) ** 2, 0)

    theta = np.linspace(0, 2 * np.pi, 200)
    circle_x = 3.0 + 1.4 * np.cos(theta)
    circle_y = 2.0 + 1.4 * np.sin(theta)

    plt.figure(figsize=(12, 5))

    plt.subplot(1, 2, 1)
    plt.plot(history)
    plt.xlabel('迭代代数')
    plt.ylabel('历史最优目标值')
    plt.title('学习引导搜索的收敛过程')
    plt.grid(alpha=0.3)

    plt.subplot(1, 2, 2)
    plt.contour(Xg, Yg, np.log1p(Z), levels=25)
    plt.scatter(archive_X[:, 0], archive_X[:, 1], s=12, alpha=0.25, label='历史样本')
    if len(guide_path) > 0:
        plt.plot(guide_path[:, 0], guide_path[:, 1], linestyle='--', linewidth=1.5, label='学习引导中心')
    plt.plot(circle_x, circle_y, color='red', linewidth=1.5, label='禁区边界')
    plt.scatter(best_x[0], best_x[1], marker='*', s=180, label='当前最优')
    plt.xlabel('x')
    plt.ylabel('y')
    plt.title('机器学习增强的搜索轨迹')
    plt.grid(alpha=0.3)
    plt.legend()

    plt.tight_layout()
    plt.show()


if __name__ == '__main__':
    best_x, best_y, history, archive_X, guide_path = learning_guided_ea()
    print('best_x =', best_x)
    print('best_y =', best_y)
    plot_result(best_x, history, archive_X, guide_path)
```

### 结果解读

运行后可以看到：

1. 左图展示历史最优值如何逐步下降；
2. 右图展示历史样本、禁区位置以及学习引导中心的移动轨迹。

这个例子强调的重点是：

- 学习模型没有代替真实目标函数；
- 它只是帮算法更快聚焦到更有希望的区域；
- 这正是“机器学习增强元启发式”的典型思路之一。

## 机器学习增强元启发式的优缺点及适用场景

### 优点

- 能利用历史数据提炼搜索规律；
- 可以减少无效搜索，提高搜索效率；
- 可用于参数控制、可行性预测、初始化、候选筛选等多个环节；
- 和现有元启发式算法兼容性通常较好。

### 局限

- 学习模型质量不好时，可能误导搜索；
- 额外引入了建模和调参成本；
- 不同问题适合的学习模块差异很大；
- 目前更像一个方法方向，而不是一个统一固定模板。

### 适用场景

- 历史搜索数据可积累的问题；
- 可行域复杂、盲搜代价高的问题；
- 需要动态调参或策略选择的问题；
- 想在传统元启发式上进一步提升效率的研究场景。

## 一句话总结

机器学习增强的元启发式，本质上是在给传统启发式算法加上一层“边搜边学”的能力，让搜索不只是试错，还能逐步积累经验。
