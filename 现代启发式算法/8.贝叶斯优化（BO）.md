# 贝叶斯优化（BO）

贝叶斯优化（Bayesian Optimization, BO）是一种非常适合**昂贵黑箱函数优化**的方法。

这里所谓“昂贵”，不是指代码长，而是指一次目标函数评估本身就很贵，比如：

- 跑一次仿真要 30 分钟；
- 做一次物理实验要耗材料；
- 训练一次模型要花很多算力；
- 调一次工艺参数要停线测试。

在这种场景下，你已经不是在追求“每代可以多试几个点”，而是在追求“每一次试验都尽量值钱”。

这就是贝叶斯优化擅长的地方。

## 问题定义

考虑如下黑箱优化问题：

$$
\min_{x \in \Omega} f(x)
$$

其中：

- $f(x)$ 没有解析表达式可用；
- 梯度通常不可得；
- 但给定一个 $x$，可以通过实验、仿真或程序运行得到函数值 $f(x)$。

贝叶斯优化不会盲目试点，而是先对未知函数建立一个概率模型，最常见的是高斯过程（Gaussian Process, GP）：

$$
f(x) \sim \mathcal{GP}(m(x), k(x, x'))
$$

其中：

- $m(x)$ 是均值函数；
- $k(x,x')$ 是核函数。

## 核心机制

BO 的核心可以概括成两步：

1. 用代理模型描述“我当前认为函数大概长什么样”；
2. 用采集函数决定“下一次最值得试哪里”。

### 1. 高斯过程后验

给定已观测样本集：

$$
\mathcal{D}_t = \{(x_i, y_i)\}_{i=1}^t
$$

高斯过程在新点 $x$ 处的后验均值和后验方差为：

$$
\mu_t(x) = k(x, X)^T (K + \sigma_n^2 I)^{-1} y
$$

$$
\sigma_t^2(x) = k(x,x) - k(x,X)^T (K + \sigma_n^2 I)^{-1} k(x,X)
$$

其中：

- $K$ 是训练样本之间的核矩阵；
- $\sigma_n^2$ 是噪声项。

后验均值表示“预测值”，后验方差表示“不确定性”。

### 2. 采集函数

为了决定下一次试验点，常用一个采集函数。以最简单的下置信界（LCB）为例，在最小化问题中常写成：

$$
a_t(x) = \mu_t(x) - \kappa \sigma_t(x)
$$

然后选择：

$$
x_{t+1} = \arg\min_{x \in \Omega} a_t(x)
$$

这个式子很有意思：

- $\mu_t(x)$ 小，说明模型觉得这里可能值不错；
- $\sigma_t(x)$ 大，说明模型对这里还不确定；
- 两者结合起来，就体现了“开发”和“探索”的平衡。

## 人话

你可以把贝叶斯优化理解成一个“会做笔记的试验员”。

普通随机搜索像是随便调温度乱试；而贝叶斯优化更像：

1. 每做完一次实验，就画一张“我觉得函数大概长这样”的草图；
2. 同时标记出“哪些地方看起来好”“哪些地方我其实还没搞清楚”；
3. 下一次优先去那个“值得试”的位置。

所以 BO 的重点不在于一次试很多，而在于**每次都尽量试得聪明**。

## 基本流程

### 初始化

- 先选少量初始样本点；
- 真实评估这些点；
- 建立初始高斯过程模型。

### 对每一轮重复以下步骤

#### 第一步：拟合后验模型

根据已有数据更新高斯过程后验分布。

#### 第二步：优化采集函数

寻找当前最值得评估的新点。

#### 第三步：真实评估

对新点调用真实黑箱函数，得到新观测值。

#### 第四步：更新数据集

把新样本加入历史数据。

#### 第五步：判断是否终止

例如达到预算上限或已经找到满意解。

## 建模例题：化学反应温度寻优

### 问题背景

假设某化学反应的温度设置会影响副产物损失。我们希望找到一个最佳温度，使损失尽量小。

但每做一次实验都很贵，因此不能像普通优化那样密集扫描所有温度。

这个问题非常适合 BO：

- 变量维数低；
- 每次评估代价高；
- 需要尽量少试但试得更准。

### Python 代码

```python
import math
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimSun']
plt.rcParams['axes.unicode_minus'] = False


def expensive_objective(x):
    return (
        0.015 * (x - 86) ** 2
        - 2.4 * math.exp(-((x - 92) ** 2) / 70)
        + 0.35 * math.sin(x / 4)
    )


def rbf_kernel(x1, x2, length_scale=7.0, sigma_f=1.2):
    x1 = np.asarray(x1).reshape(-1, 1)
    x2 = np.asarray(x2).reshape(-1, 1)
    sqdist = (x1 - x2.T) ** 2
    return sigma_f ** 2 * np.exp(-0.5 * sqdist / (length_scale ** 2))


def gp_posterior(x_train, y_train, x_star, noise=1e-6):
    K = rbf_kernel(x_train, x_train) + noise * np.eye(len(x_train))
    K_s = rbf_kernel(x_train, x_star)
    K_ss = rbf_kernel(x_star, x_star) + 1e-8 * np.eye(len(x_star))

    L = np.linalg.cholesky(K)
    alpha = np.linalg.solve(L.T, np.linalg.solve(L, y_train))

    mu = K_s.T @ alpha
    v = np.linalg.solve(L, K_s)
    cov = K_ss - v.T @ v
    std = np.sqrt(np.maximum(np.diag(cov), 1e-12))

    return mu, std


def acquisition_lcb(mu, std, kappa=2.0):
    return mu - kappa * std


def bayesian_optimization(bounds, n_init=3, n_iter=15):
    x_train = np.array([45.0, 80.0, 115.0])
    y_train = np.array([expensive_objective(x) for x in x_train])

    history = [np.min(y_train)]

    for _ in range(n_iter):
        x_grid = np.linspace(bounds[0], bounds[1], 500)
        mu, std = gp_posterior(x_train, y_train, x_grid)
        acq = acquisition_lcb(mu, std, kappa=2.0)

        order = np.argsort(acq)
        x_next = None
        for idx in order:
            cand = x_grid[idx]
            if np.min(np.abs(x_train - cand)) > 0.8:
                x_next = cand
                break

        if x_next is None:
            x_next = x_grid[order[0]]

        y_next = expensive_objective(x_next)
        x_train = np.append(x_train, x_next)
        y_train = np.append(y_train, y_next)
        history.append(np.min(y_train))

    best_idx = np.argmin(y_train)
    return x_train, y_train, x_train[best_idx], y_train[best_idx], history


def plot_result(x_train, y_train, best_x, best_y, history, bounds):
    x_grid = np.linspace(bounds[0], bounds[1], 500)
    true_y = np.array([expensive_objective(x) for x in x_grid])
    mu, std = gp_posterior(x_train, y_train, x_grid)

    plt.figure(figsize=(12, 5))

    plt.subplot(1, 2, 1)
    plt.plot(x_grid, true_y, label='真实目标函数', linewidth=1.5)
    plt.plot(x_grid, mu, label='GP 后验均值', linewidth=1.5)
    plt.fill_between(x_grid, mu - 1.96 * std, mu + 1.96 * std, alpha=0.25, label='95%置信区间')
    plt.scatter(x_train, y_train, s=45, label='已评估样本')
    plt.scatter([best_x], [best_y], marker='*', s=180, label='当前最优')
    plt.xlabel('温度')
    plt.ylabel('损失')
    plt.title('贝叶斯优化中的代理模型')
    plt.grid(alpha=0.3)
    plt.legend()

    plt.subplot(1, 2, 2)
    plt.plot(history, marker='o')
    plt.xlabel('评估轮次')
    plt.ylabel('历史最优损失')
    plt.title('贝叶斯优化收敛过程')
    plt.grid(alpha=0.3)

    plt.tight_layout()
    plt.show()


if __name__ == '__main__':
    bounds = (40, 120)
    x_train, y_train, best_x, best_y, history = bayesian_optimization(bounds)

    print('最优温度 =', best_x)
    print('最优损失 =', best_y)
    plot_result(x_train, y_train, best_x, best_y, history, bounds)
```

### 结果解读

运行后通常能看到：

1. 左图里，高斯过程给出了当前对未知函数的“猜测”和“不确定性”；
2. 右图里，随着评估次数增加，历史最优值逐步下降。

这个例子最能体现 BO 的价值：**在试验预算很少的情况下，依然尽可能高效地逼近最优区域。**

## 贝叶斯优化的优缺点及适用场景

### 优点

- 样本效率很高；
- 很适合昂贵目标函数；
- 能显式处理不确定性；
- 探索与开发的平衡比较自然。

### 局限

- 更适合低维到中低维问题；
- 高维时高斯过程建模会变困难；
- 采集函数自身也要优化；
- 如果评估很便宜，BO 的额外建模开销反而未必划算。

### 适用场景

- 超参数优化；
- 仿真参数调优；
- 工艺实验优化；
- 物理实验设计；
- 小样本昂贵黑箱优化。

## 一句话总结

贝叶斯优化的核心，不是“试很多次”，而是**把每一次试验都尽量变成高价值试验**。
