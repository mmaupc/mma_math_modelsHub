# 代理模型与昂贵优化（SAEA）

代理模型辅助进化算法（Surrogate-Assisted Evolutionary Algorithms, SAEA）主要针对这样一种场景：

- 目标函数评估极其昂贵；
- 但又不能只做局部调参；
- 仍然需要保留进化算法那种全局搜索能力。

它的核心思想非常直接：

1. 先用少量真实样本训练一个便宜得多的代理模型；
2. 再让进化算法优先在代理模型上“多想、多试”；
3. 只把少数真正值得的候选解送去做昂贵的真实评估。

所以它不是完全替代进化算法，而是给进化算法配了一个“便宜的练习场”。

## 问题定义

我们仍然考虑黑箱优化问题：

$$
\min_{x \in \Omega} f(x)
$$

但这里额外强调一件事：一次真实评估 $f(x)$ 的代价很高。

于是我们引入代理模型：

$$
\hat{f}_t(x) = s(x; \mathcal{D}_t)
$$

其中：

- $\mathcal{D}_t = \{(x_i, f(x_i))\}_{i=1}^{n_t}$ 是截至第 $t$ 轮的真实样本集；
- $s(\cdot)$ 是代理模型，比如 RBF、Kriging、SVR、神经网络等。

## 核心机制

SAEA 的关键不只是“拟合一个代理”，而是如何用代理帮助搜索。

### 1. 代理建模

以 RBF（径向基函数）模型为例，常见形式可写成：

$$
\hat{f}(x) = \sum_{i=1}^{n} w_i \phi(\|x-x_i\|) + b_0 + \sum_{j=1}^{D} b_j x_j
$$

这里：

- $\phi(\cdot)$ 是基函数；
- $x_i$ 是已有样本；
- $w_i, b_j$ 由已知样本拟合得到。

### 2. 代理上的候选搜索

进化算法先在代理模型上寻找看起来不错的点，而不是一上来就做真实评估。

为了避免只在代理模型最优附近打转，常常会引入一个兼顾预测值与多样性的准则，例如：

$$
U_t(x) = \hat{f}_t(x) - \lambda \cdot \min_{x_i \in \mathcal{D}_t} \|x - x_i\|
$$

其中：

- $\hat{f}_t(x)$ 小，说明代理预测这里可能不错；
- 距离历史样本远，说明这里更有探索价值；
- $\lambda$ 控制探索与开发的平衡。

### 3. 有限次真实评估

最后只选少量候选点做真实评估，再把新结果补回样本集。

这使得算法把大量“便宜尝试”留在代理模型上，把稀缺的真实预算用在更值得的地方。

## 人话

如果真实仿真像“做一次风洞实验”，那你肯定不能每想到一个新方案就直接上风洞。

更现实的做法是：

1. 先用已有实验数据训练一个近似模型；
2. 用这个近似模型先筛掉一大堆明显不行的方案；
3. 最后只把少数最值得怀疑、最值得期待的方案送去做真实验。

这就是代理模型辅助优化的核心逻辑。

## 基本流程

### 初始化

- 在可行域内采样少量初始点；
- 对这些点做真实评估；
- 训练初始代理模型。

### 对每一轮重复以下步骤

#### 第一步：更新代理模型

用当前真实样本集重新拟合代理模型。

#### 第二步：在代理上搜索

利用进化算法或其他策略，在代理模型上生成并筛选候选解。

#### 第三步：选择待真实评估点

根据预测值、多样性或不确定性选出少量点。

#### 第四步：做真实评估

对这些点调用昂贵目标函数。

#### 第五步：更新样本集

把新点加入真实样本集，刷新最优解。

## 建模例题：昂贵二维函数的代理辅助优化

### 问题背景

我们用 Branin 函数来模拟一个“昂贵但可评估”的二维黑箱问题：

$$
f(x,y) = \left(y - \frac{5.1}{4\pi^2}x^2 + \frac{5}{\pi}x - 6\right)^2 + 10\left(1-\frac{1}{8\pi}\right)\cos x + 10
$$

之所以选这个例子，是因为：

- 它是二维连续变量问题；
- 有多个局部低谷；
- 很适合展示“少量真实评估 + 代理辅助搜索”的流程。

### Python 代码

```python
import random
import math
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimSun']
plt.rcParams['axes.unicode_minus'] = False


def branin(x):
    x1, x2 = x
    a = 1.0
    b = 5.1 / (4 * math.pi ** 2)
    c = 5 / math.pi
    r = 6.0
    s = 10.0
    t = 1 / (8 * math.pi)
    return a * (x2 - b * x1 ** 2 + c * x1 - r) ** 2 + s * (1 - t) * math.cos(x1) + s


def latin_hypercube(n_samples, lower, upper, seed=42):
    np.random.seed(seed)
    dim = len(lower)
    result = np.zeros((n_samples, dim))

    for j in range(dim):
        perm = np.random.permutation(n_samples)
        result[:, j] = (perm + np.random.rand(n_samples)) / n_samples
        result[:, j] = lower[j] + result[:, j] * (upper[j] - lower[j])

    return result


def fit_rbf(X, y, reg=1e-8):
    n = len(X)
    dim = X.shape[1]

    Phi = np.zeros((n, n))
    for i in range(n):
        for j in range(n):
            r = np.linalg.norm(X[i] - X[j])
            Phi[i, j] = r ** 3

    P = np.hstack([np.ones((n, 1)), X])
    A = np.block([
        [Phi + reg * np.eye(n), P],
        [P.T, np.zeros((dim + 1, dim + 1))]
    ])
    rhs = np.concatenate([y, np.zeros(dim + 1)])

    coef = np.linalg.solve(A, rhs)
    w = coef[:n]
    b = coef[n:]
    return w, b


def rbf_predict(x, X, w, b):
    phi = np.array([np.linalg.norm(x - xi) ** 3 for xi in X])
    return np.dot(w, phi) + b[0] + np.dot(b[1:], x)


def de_candidates(pop, lower, upper, F=0.7, CR=0.9):
    pop_size, dim = pop.shape
    cands = []

    for i in range(pop_size):
        ids = list(range(pop_size))
        ids.remove(i)
        r1, r2, r3 = random.sample(ids, 3)

        v = pop[r1] + F * (pop[r2] - pop[r3])
        v = np.clip(v, lower, upper)

        u = pop[i].copy()
        j_rand = random.randrange(dim)
        for j in range(dim):
            if random.random() < CR or j == j_rand:
                u[j] = v[j]

        cands.append(u)

    return np.array(cands)


def surrogate_assisted_de(budget=35, init_samples=16, generations_on_surrogate=4, seed=42):
    np.random.seed(seed)
    random.seed(seed)

    lower = np.array([-5.0, 0.0])
    upper = np.array([10.0, 15.0])

    X = latin_hypercube(init_samples, lower, upper, seed=seed)
    y = np.array([branin(x) for x in X])

    best_idx = np.argmin(y)
    best_x = X[best_idx].copy()
    best_y = y[best_idx]
    history = [best_y]

    pop = X[np.argsort(y)[:init_samples]].copy()

    while len(y) < budget:
        w, b = fit_rbf(X, y)
        pop_curr = pop.copy()

        for _ in range(generations_on_surrogate):
            offspring = de_candidates(pop_curr, lower, upper)
            pool = np.vstack([pop_curr, offspring])

            scores = []
            for cand in pool:
                pred = rbf_predict(cand, X, w, b)
                dist = np.min(np.linalg.norm(X - cand, axis=1))
                score = pred - 0.2 * dist
                scores.append(score)

            ids = np.argsort(scores)[:len(pop_curr)]
            pop_curr = pool[ids]

        x_new = pop_curr[0].copy()
        if np.min(np.linalg.norm(X - x_new, axis=1)) < 1e-3:
            x_new = lower + np.random.rand(2) * (upper - lower)

        y_new = branin(x_new)

        X = np.vstack([X, x_new])
        y = np.append(y, y_new)
        pop = pop_curr

        if y_new < best_y:
            best_x = x_new.copy()
            best_y = y_new

        history.append(best_y)

    return best_x, best_y, history, X, y


def plot_result(best_x, history, X):
    xs = np.linspace(-5, 10, 220)
    ys = np.linspace(0, 15, 220)
    Xg, Yg = np.meshgrid(xs, ys)
    Z = (
        (Yg - 5.1 / (4 * math.pi ** 2) * Xg ** 2 + 5 / math.pi * Xg - 6) ** 2
        + 10 * (1 - 1 / (8 * math.pi)) * np.cos(Xg)
        + 10
    )

    plt.figure(figsize=(12, 5))

    plt.subplot(1, 2, 1)
    plt.plot(range(len(history)), history, marker='o')
    plt.xlabel('真实评估次数阶段')
    plt.ylabel('历史最优目标值')
    plt.title('代理辅助优化收敛过程')
    plt.grid(alpha=0.3)

    plt.subplot(1, 2, 2)
    plt.contour(Xg, Yg, Z, levels=25)
    plt.scatter(X[:, 0], X[:, 1], s=28, alpha=0.6, label='真实评估点')
    plt.scatter(best_x[0], best_x[1], marker='*', s=180, label='当前最优')
    plt.xlabel('x')
    plt.ylabel('y')
    plt.title('Branin 函数上的采样分布')
    plt.grid(alpha=0.3)
    plt.legend()

    plt.tight_layout()
    plt.show()


if __name__ == '__main__':
    best_x, best_y, history, X, y = surrogate_assisted_de()
    print('best_x =', best_x)
    print('best_y =', best_y)
    plot_result(best_x, history, X)
```

### 结果解读

运行后可以看到：

1. 左图展示随着真实评估预算增加，历史最优值逐步下降；
2. 右图展示真实评估点如何分布在搜索空间里，以及最终最优点落在哪里。

这个例子最重要的不是“找到全局最优多快”，而是体会：**很多候选尝试其实是在代理模型上完成的，真实预算被节省下来了。**

## 代理模型辅助优化的优缺点及适用场景

### 优点

- 当真实评估很贵时，能够显著节省预算；
- 能保留进化算法的全局搜索能力；
- 代理模型种类灵活，可按问题替换；
- 特别适合仿真、实验驱动优化。

### 局限

- 代理模型本身可能有误差；
- 代理误差如果积累，可能把搜索带偏；
- 需要额外设计“什么时候真评估、选谁真评估”；
- 实现复杂度明显高于普通进化算法。

### 适用场景

- 昂贵仿真优化；
- 实验驱动建模优化；
- 飞行器/结构参数设计；
- 工艺实验调优；
- 真实评估预算特别紧张的连续优化问题。

## 一句话总结

代理模型辅助优化的核心，是用一个便宜模型帮你“多想几步”，再把稀缺的真实预算花在更值得的点上。
