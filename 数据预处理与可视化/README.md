# 数据预处理与可视化

## 项目简介

本项目围绕数据分析中的基础但关键的三个环节展开：缺失值处理、特征缩放和数据可视化。学习者将通过多个 Notebook 和配套数据集，理解数据预处理背后的数学原理，并掌握将原始数据转化为可分析、可建模数据的完整思路。

## 目录结构

- `1.处理缺失值Project1.ipynb`：演示如何识别缺失值、统计缺失比例，并尝试基础填补方法。
- `2.归一化与标准化Project2.ipynb`：介绍常见的特征缩放方法，比较归一化与标准化的差异和适用场景。
- `3.数据可视化Project3.ipynb`：展示直方图、箱线图等常见图表，帮助理解数据分布和异常波动。
- `4.案例Project4.ipynb`：将前面的预处理知识整合到一个完整案例中，训练真实分析流程。
- `5.练习.ipynb`：供学生独立练习，巩固缺失值处理、特征缩放与可视化技能。
- `Examples.csv`：课堂示例数据，可用于演示成绩数据中的缺失值与异常值问题。
- `tax_data.csv`：税收时间序列案例数据，用于完整展示缺失值处理、标准化和可视化流程。
- `train.csv`：Titanic 训练集，适合做更综合的数据清洗、特征工程和建模前处理练习。
- `test.csv`：Titanic 测试集，可用于练习将同样的预处理流程迁移到新数据上。
- `end_to_end_workflow.py`：本项目补充的工业级整合脚本，用于将读取、清洗、标准化、可视化和导出串联成完整流程。

## 核心理论与数学方法

### 1. 缺失值处理

在真实业务中，缺失值几乎不可避免。例如，税收数据可能因为上报延迟、人工录入遗漏或历史资料不完整而出现空值。如果不先处理缺失值，后续的均值计算、图表展示甚至模型训练都可能偏离真实情况。

常见的基础方法有两种：

- **均值填充（Mean Imputation）**：用该列已有数值的平均值填补缺失项。
- **中位数填充（Median Imputation）**：用排序后位于中间位置的数值填补缺失项。

它们背后的逻辑可以这样理解：

- 当数据整体比较平稳、没有特别极端的异常值时，均值能够代表整体水平，因此均值填充更自然。
- 当数据中存在明显偏态或极端值时，中位数更稳健，不容易被异常值拉高或拉低，因此中位数填充更可靠。

从业务角度看，若某个月税收记录缺失：

- 用均值填充，相当于用“整体平均税收水平”做近似估计；
- 用中位数填充，相当于用“典型月份的税收水平”做近似估计。

### 2. 归一化（Min-Max Scaling）

归一化的目标是把不同量纲、不同取值范围的数据压缩到统一尺度，最典型的是映射到区间 $[0, 1]$。其公式为：

$$
x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}
$$

其中：

- $x$ 是原始值；
- $x_{\min}$ 是该特征的最小值；
- $x_{\max}$ 是该特征的最大值；
- $x'$ 是归一化后的结果。

它的作用可以概括为以下三点：

- 把不同单位、不同量级的特征放到同一个比较尺度下；
- 避免数值范围特别大的变量在距离计算中“压制”其他变量；
- 让很多算法的训练过程更稳定。

例如，在业务分析中，“税收金额”和“人口数量”可能量级完全不同。如果直接送入算法，数值更大的特征往往会主导结果。归一化之后，每个特征都被压到 $0$ 到 $1$ 之间，更便于公平比较。

归一化常见于以下场景：

- KNN、K-Means 这类依赖距离的算法；
- 神经网络输入特征预处理；
- 明确希望数值落在固定范围内的任务。

需要注意的是，归一化非常依赖最大值和最小值，因此**对异常值较敏感**。如果某一列中出现极端值，很多正常样本可能会被挤压到很小的区间内。

### 3. 标准化（Z-score Standardization）

标准化的目标不是把数据压到固定区间，而是将数据转换为“以均值为中心、以标准差为尺度”的形式。其公式为：

$$
z = \frac{x - \mu}{\sigma}
$$

其中：

- $x$ 是原始值；
- $\mu$ 是均值（Mean）；
- $\sigma$ 是标准差（Standard Deviation）；
- $z$ 是标准化后的结果。

如果进一步从统计学角度理解，均值与标准差可以写成：

$$
\mu = \frac{1}{n} \sum_{i=1}^{n} x_i
$$

$$
\sigma = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (x_i - \mu)^2}
$$

这里：

- **均值 $\mu$** 描述数据的中心位置，也可以理解为样本整体的平均水平；
- **标准差 $\sigma$** 描述数据围绕均值波动的程度。标准差越大，说明数据越分散；标准差越小，说明数据越集中。

标准化之后，数据通常会具有以下特征：

- 平均值接近 $0$；
- 标准差接近 $1$；
- 不同特征之间更容易比较“偏离平均水平的程度”。

例如，若某个月税收的标准化结果为 $z = 2$，可以理解为它大约高于平均税收水平两个标准差，说明该月税收明显偏高；若 $z = -1.5$，则说明它低于平均水平约 1.5 个标准差。

为什么标准化通常比归一化更不容易受到异常值的严重影响？

- 归一化直接由 $x_{\min}$ 和 $x_{\max}$ 决定缩放边界，只要出现一个极端值，就可能明显改变映射结果；
- 标准化依赖的是整体均值和整体波动程度，不是只盯住两个端点；
- 因此，相比 Min-Max Scaling，标准化在很多场景下通常更稳健。

不过需要说明的是，标准化并不是完全不受异常值影响。极端值依然会影响均值和标准差，只是相较于“完全依赖最大值和最小值”的归一化方法，它在实际业务中往往更适合作为通用预处理方案。

标准化常见于以下场景：

- 线性回归、逻辑回归；
- 支持向量机（SVM）；
- 主成分分析（PCA）；
- 希望比较样本相对偏离程度的聚类和异常检测任务。

## 环境依赖

运行本项目中的 Notebook 与整合脚本，通常需要以下 Python 库：

- `pandas`：用于读取和处理表格数据。
- `numpy`：用于基础数值计算。
- `scikit-learn`：用于缺失值填补、标准化等预处理操作。
- `matplotlib`：用于基础图表绘制。
- `seaborn`：用于更美观的统计可视化。

建议使用以下命令安装依赖：

```bash
pip install pandas numpy scikit-learn matplotlib seaborn jupyter
```

如果你希望直接运行整合后的工作流脚本，可以在当前目录执行：

python end_to_end_workflow.py

## 学习建议

建议按照以下顺序学习：

1. 先理解为什么原始数据不能直接用于分析或建模；
2. 再掌握缺失值处理与特征缩放的数学含义；
3. 最后通过可视化验证预处理是否真正改善了数据质量。

如果你是初学者，可以把本项目理解为一次完整的“数据进入模型前的准备训练”：先把数据清理干净，再让不同特征变得可比较，最后用图表检查处理结果是否合理。
